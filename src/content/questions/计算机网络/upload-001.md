## 🧩 题目

前端向后端上传图片的一般步骤是什么？对于大文件上传，如何实现分片上传和断点续传？

---

## 💡 解题思路

1. 普通上传流程：选择文件 → 预览/校验 → 构造 FormData 或直传对象存储 → 显示进度
2. 分片上传：File.slice、并发上传分片、服务端合并
3. 断点续传：文件唯一标识（hash）、已上传分片记录、续传只补齐缺失分片
4. 简要说明安全和体验：大小/类型校验、进度反馈、重试机制

---

## ✅ 面试回答（标准版）

1. **普通图片上传的完整步骤**  
   - **前端选择文件**：  
     - 使用 `<input type="file">` 或拖拽区域获取 `File` 对象。  
   - **前端校验与预览（可选）**：  
     - 校验类型（`image/*`）、大小限制、数量限制等。  
     - 使用 `URL.createObjectURL` 或 `FileReader` 做本地预览。  
   - **构造请求并上传**：  
     - 方式一：使用 `FormData` 携带文件：  

     ```js
     const form = new FormData();
     form.append('file', file);
     await fetch('/upload', { method: 'POST', body: form });
     ```

     - 方式二：直传到对象存储（OSS、S3 等），先从业务服务获取带签名的上传 URL，再 `PUT` 到对象存储。  
   - **显示上传进度**：  
     - XHR 可以通过 `xhr.upload.onprogress`，Fetch 可结合 Streams 或使用第三方库。  
   - **处理响应与保存 URL**：  
     - 后端返回图片 URL 或资源 ID，前端保存到业务数据中。  

2. **大文件分片上传的基本思路**  
   - **为什么需要分片**：  
     - 大文件一次性上传容易超时、中断且失败后需重头再来；分片可以并发上传、失败重传单个分片。  
   - **核心步骤**：  
     1）在前端使用 `File.slice` 将文件按固定大小（如 2MB/5MB）切成多个分片：  

     ```js
     const chunkSize = 2 * 1024 * 1024;
     const chunks = [];
     for (let start = 0; start < file.size; start += chunkSize) {
       const end = Math.min(start + chunkSize, file.size);
       chunks.push(file.slice(start, end));
     }
     ```

     2）为每个分片生成索引（chunkIndex）并发起上传，请求体中包含：文件标识、分片序号、总分片数等。  
     3）后端将每个分片暂存（本地磁盘或对象存储）并记录上传状态。  
     4）所有分片上传完成后，前端通知后端进行「合并」操作（服务器按顺序拼接分片）。  
   - **前端可做的优化**：  
     - 设置并发上传数量限制（如一次 3~5 个分片）  
     - 对失败的分片进行重试  

3. **断点续传的关键点**  
   - **文件唯一标识**：  
     - 通常通过计算文件的 hash（例如 MD5、SHA-1）或「文件名 + 大小 + 最后修改时间」等方式生成一个 fileId，用于识别同一文件。  
   - **记录已上传分片**：  
     - 前端在开始上传前先向后端查询该 fileId 的上传记录，获取「已完成的分片列表」。  
     - 前端只上传未完成的分片，实现断点续传。  
   - **本地持久化**（可选）：  
     - 可以把 fileId 与分片上传进度记录在 localStorage/IndexedDB 中，配合服务端记录实现跨重启、跨页面的续传。  

4. **安全与体验上的补充**  
   - 前端应对文件类型、大小做预校验，并给予清晰的错误提示。  
   - 对上传过程增加进度条、取消/暂停按钮，提高用户体验。  
   - 后端需要限制单文件/单用户上传频率与大小，并对文件内容做安全检查（如病毒扫描、黑名单类型过滤等）。  

一句话：小文件可以直接用 FormData 一次性上传；大文件则应基于 `File.slice` 实现分片上传，并通过「文件唯一标识 + 已上传分片记录」实现断点续传和失败重传。
