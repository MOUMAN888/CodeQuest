## 🧩 题目

你的项目通常需要兼容哪些机型 / 分辨率？在不同机型上出现样式不一致或布局错乱的常见原因有哪些？你一般如何排查和解决这些兼容性问题？

---

## 💡 解题思路

1. 给出典型兼容范围：主流 iOS/Android 机型、PC 分辨率
2. 不一致原因：不同 DPR、浏览器内核、系统字体、像素密度、刘海/安全区域等
3. 解决：响应式布局、媒体查询、rem/viewport、调试工具、多机型真机联调

---

## ✅ 面试回答（标准版）

1. **常见兼容范围示例**  
   - 移动端：  
     - iOS：iPhone 8 及以上（包括刘海屏，如 iPhone X/12/13 系列）。  
     - Android：常见 720p/1080p/2K 分辨率、不同厂商浏览器与 WebView。  
   - PC 端：  
     - 常见分辨率 1366×768、1440×900、1920×1080 及更高，兼容最新版 Chrome/Edge/Firefox/Safari。  

2. **样式不一致的常见原因**  
   - **布局与单位问题**：  
     - 使用绝对像素而非弹性布局，导致在小屏或大屏上溢出或留白。  
     - 忽视 `devicePixelRatio` 和缩放比例，导致字体和元素显示异常。  
   - **不同浏览器内核与默认样式差异**：  
     - 不同内核（Blink/WebKit/Gecko）对某些 CSS 特性支持程度不同。  
     - 未统一重置默认样式（如 `margin`、`line-height`）导致不同浏览器渲染差异。  
   - **字体与行高**：  
     - 系统字体不一致、字体回退策略不同，导致文本换行位置和高度不同。  
   - **特殊机型特性**：  
     - 刘海屏、安全区域（safe area）、导航栏遮挡。  
     - Android 某些定制浏览器对 `vh`、固定定位等支持存在 bug。  

3. **排查与解决思路**  
   - **使用调试工具与真机调试**：  
     - Chrome DevTools 的设备模拟器 + 远程调试移动设备（Chrome/Safari）。  
     - 小程序则使用各平台开发者工具 + 真机预览对比。  
   - **统一基础样式与响应式设计**：  
     - 使用 CSS Reset / Normalize 统一默认样式。  
     - 使用弹性布局（Flex、Grid）、百分比宽度、`rem` / `vw` 等做响应式。  
     - 针对极端宽/窄屏使用媒体查询做边界适配。  
   - **针对已知问题做兼容性处理**：  
     - 为刘海屏增加安全区域 padding（利用 `env(safe-area-inset-*)`）。  
     - 避免依赖兼容性较差的 CSS 特性或提供降级方案。  

4. **总结**  
   - 样式兼容性问题本质上是「不同环境下渲染实现差异」与「布局设计不够弹性」共同导致的。  
   - 通过合理的响应式布局、统一基础样式、真机调试和有针对性的兼容处理，可以大幅降低不同机型之间的视觉差异。
