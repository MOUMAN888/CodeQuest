## 🧩 题目

前端性能优化通常可以从哪些维度入手？结合你的实习或项目经历，举例说明你做过哪些具体的性能优化措施。

---

## 💡 解题思路

1. 维度：加载性能（网络）、渲染性能（渲染/动画）、运行时性能（JS 执行）、交互体验、感知性能
2. 指标与工具：Lighthouse、Performance 面板、Web Vitals、监控埋点
3. 示例优化措施：按需加载、压缩资源、缓存策略、减少重排重绘、长任务切分等

---

## ✅ 面试回答（标准版）

回答这类问题时，先列出「优化维度框架」，再挑 2~3 个你实际做过的高价值案例展开即可。

1. **前端性能优化的常见维度**  
   - **加载性能**：  
     - 减少首屏资源体积与请求数（代码分割、按需加载、压缩、Tree Shaking）。  
     - 使用 HTTP 缓存、CDN 加速、预加载/预获取关键资源。  
   - **渲染性能**：  
     - 减少回流重绘，使用 `transform`/`opacity` 做动画。  
     - 避免过度嵌套的 DOM 与复杂布局，使用虚拟列表等技术。  
   - **运行时性能（JS 执行）**：  
     - 减少长任务、切分耗时计算、使用 Web Worker。  
     - 优化监听事件、防抖/节流、高频渲染组件的 memo 化。  
   - **交互与感知性能**：  
     - 提供 Skeleton（骨架屏）、loading 反馈，优化首屏 FCP/LCP。  
     - 减少输入/滚动卡顿，使用合适的事件绑定与节流。  

2. **项目中的具体优化案例示例（可按需替换为你的实际经历）**  
   - **案例一：首屏加载优化**  
     - 问题：首页 bundle 过大，首屏加载时间 > 4s。  
     - 措施：  
       1）使用打包分析工具（如 `webpack-bundle-analyzer`）找出体积大户（某 UI 库整体引入、moment 等）。  
       2）将路由改为懒加载，拆分按路由维度的异步 chunk。  
       3）对 UI 库改为按需引入，替换掉部分重型第三方库（如 `moment` → `dayjs`）。  
       4）开启 gzip/br 压缩，并配置浏览器缓存头。  
     - 效果：首页 JS 体积降低约 30~50%，Lighthouse 的 LCP/FCP 明显改善。  

   - **案例二：长列表渲染卡顿优化**  
     - 问题：一次性渲染上千条 DOM，滚动和交互明显掉帧。  
     - 措施：  
       1）引入虚拟列表（只渲染可视区域 + 缓冲区），根据滚动位置动态复用 DOM。  
       2）对列表项组件使用 `memo`/`PureComponent` 避免不必要的重复渲染。  
     - 效果：渲染节点数下降到几十个，FPS 明显提升，滚动流畅。  

   - **案例三：运行时卡顿排查与优化**  
     - 问题：某交互操作导致页面短暂冻结。  
     - 措施：  
       1）使用浏览器 Performance 面板录制性能分析，发现某次点击触发了数百毫秒的同步计算。  
       2）将耗时计算拆分为多个小任务，结合 `requestIdleCallback` / `setTimeout` 分批执行，或迁移到 Web Worker 中处理。  
     - 效果：主线程长任务被拆分，交互卡顿明显缓解。  

3. **总结**  
   - 性能优化不是单点技巧，而是围绕「加载、渲染、执行、交互」四个方向的系统性工作。  
   - 面试中要强调你**如何发现问题（监控/工具）→ 分析原因 → 制定方案 → 验证效果**的完整闭环，而不仅仅是罗列技巧名词。
