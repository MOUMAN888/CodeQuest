## 🧩 题目

JavaScript 中 `null` 和 `undefined` 的本质区别是什么？各自适合用在什么场景？

---

## 💡 解题思路

1. 从含义上区分：`undefined` 表示「未定义」，`null` 表示「有意的空」
2. 从语言层面看：默认值、属性不存在、API 返回值
3. 从类型与比较行为看：`typeof`、`==` / `===`
4. 总结在工程实践中的约定用法

---

## ✅ 面试回答（标准版）

1. **语义上的核心区别**  
   - `undefined`：表示**「不存在 / 没有被赋值」**，通常是 JS 引擎自动给出的默认值  
     - 访问未声明的变量会报错，但访问已声明未经赋值的变量得到 `undefined`  
     - 对象中不存在的属性、函数没有显式 `return` 的返回值、形参未传实参等，都默认为 `undefined`  
   - `null`：表示**「有意的空值」**，通常由开发者**主动赋值**，表达「这里目前没有值，但这是一个有效位置」  
     - 常用来表示「对象为空」、「暂时没有结果」、「已清空引用」等语义  

2. **类型与比较行为**  
   - `typeof undefined === 'undefined'`  
   - `typeof null === 'object'`（历史遗留 Bug，不代表 `null` 真的是对象）  
   - 二者在「宽松相等」下被视为相等：`null == undefined // true`  
   - 但在严格相等下不同：`null === undefined // false`  

3. **常见出现场景**  
   - `undefined` 常见于：
     - 变量声明但未赋值：`let a; console.log(a); // undefined`  
     - 对象中不存在的属性：`obj.missing // undefined`  
     - 函数没有返回值：`function f() {} f(); // undefined`  
   - `null` 常见于：
     - 显式初始化一个「未来会赋值」的对象变量：`let user = null;`  
     - 主动清空引用，帮助 GC：`obj = null;`  
     - 部分 DOM API 查找失败时返回 `null`（如 `document.getElementById()`）  

4. **工程实践中的推荐约定**  
   - **`undefined` 更多由 JS 引擎产生，用来表示「缺失的值 / 未提供的值」**  
   - **`null` 建议有意识地使用，用来表示「此处应有一个值，但目前是空的」**  
   - 在接口设计与数据模型中，通常会约定：
     - 未提供/不存在：用 `undefined` 或干脆省略该字段  
     - 明确为空（例如「未选择」但字段存在）：用 `null`  

总结：`undefined` 更偏向「系统级的未定义」，`null` 更偏向「业务语义上的空」，在代码中有意识地区分两者可以让数据含义更清晰。
