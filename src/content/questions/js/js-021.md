## 🧩 题目

`async/await` 的底层原理是什么？它是如何基于 Promise 实现的？在使用时有哪些注意事项（例如错误处理、一并并发等）？

---

## 💡 解题思路

1. `async` 函数本质上返回 Promise
2. `await` 的行为：等待 Promise 结果并暂停，背后是语法糖+状态机（或 generator）转换
3. 提醒：不要在循环中滥用顺序 `await` 阻塞；错误处理要配合 `try/catch`
4. 总结与 Promise 的关系：async/await 只是写法更「同步的 Promise」

---

## ✅ 面试回答（标准版）

1. **async 函数的本质**  
   - 在声明前加上 `async` 的函数，**一定返回一个 Promise**：  

   ```js
   async function foo() {
     return 42;
   }
   foo() instanceof Promise; // true
   ```

   - 如果在 `async` 函数中返回一个普通值，相当于 `return Promise.resolve(普通值)`  
   - 如果抛出错误，相当于返回一个 `rejected` 状态的 Promise  

2. **await 的行为与底层原理**  
   - `await expr` 会对 `expr` 求值，并**等待其变为 `fulfilled` 或 `rejected` 的 Promise**：  
     - 如果 `expr` 不是 Promise，会被包装成 `Promise.resolve(expr)`  
     - 当前 `async` 函数会在此「暂停」，将后续代码放入微任务队列，在 Promise 解决后继续执行  
   - 从底层实现来看，`async/await` 可以被编译（或等价）为一个基于 `Promise` 的状态机，早期转译器（如 Babel）会用 generator + 自动执行器模拟这一行为  
   - 因此可以理解为：**`async/await` 是对 Promise 链式调用的语法糖，让异步代码写起来像同步代码**  

3. **错误处理方式**  
   - 使用 `await` 时，应搭配 `try/catch` 做错误捕获：

   ```js
   async function fetchData() {
     try {
       const res = await api.get('/data');
       return res.data;
     } catch (err) {
       // 统一处理错误
       throw err;
     }
   }
   ```

   - 或者使用 `await` + `.catch()` 的模式：  

   ```js
   const res = await api.get('/data').catch(handleError);
   ```

4. **注意事项：避免不必要的串行等待**  
   - 下面的写法会导致请求「顺序执行」，整体耗时为两者之和：

   ```js
   const a = await fetchA();
   const b = await fetchB();
   ```

   - 如果两者互不依赖，应该并发执行：

   ```js
   const [a, b] = await Promise.all([fetchA(), fetchB()]);
   ```

   - 在 `for` 循环中使用 `await`，也要考虑是否可以改为 `Promise.all` 批量并发，而不是一项项顺序等待  

5. **与 Promise 的关系总结**  
   - `async/await` 没有引入新的异步模型，只是**Promise 的语法糖**  
   - 仍然遵循事件循环和微任务执行规则  
   - 优点是让异步逻辑的控制流更接近同步代码，易读易写，特别适合复杂的多步异步流程  

一句话：`async/await` = 「基于 Promise 的语法糖 + 状态机展开」，使用时注意错误处理和并发策略，避免把本来可以并发的任务写成串行等待。
