## 🧩 题目

`typeof` 和 `instanceof` 的底层实现原理是什么？各自适用于哪些场景？有哪些局限性？

---

## 💡 解题思路

1. `typeof`：基于内部 `[[Type]]` / `Typeof` 运算，结果是字符串
2. `instanceof`：基于原型链判断，检查右侧构造函数的 `prototype` 是否在左侧对象原型链上
3. 列出典型例子：`typeof null`、`typeof function`、基本类型和引用类型
4. 分别说明适用场景与常见坑

---

## ✅ 面试回答（标准版）

1. **`typeof` 的原理与特性**  
   - `typeof` 是一个一元运算符，对操作数执行「类型标签」检测，返回一个字符串，如 `'number'`、`'string'` 等  
   - 底层基于规范里的 `Typeof` 运算，会对不同内部类型返回固定字符串：
     - `typeof 1 === 'number'`  
     - `typeof 'foo' === 'string'`  
     - `typeof true === 'boolean'`  
     - `typeof undefined === 'undefined'`  
     - `typeof Symbol() === 'symbol'`  
     - `typeof 10n === 'bigint'`  
     - `typeof function(){} === 'function'`（函数是特殊的 object，会单独返回 `'function'`）  
     - **历史遗留问题**：`typeof null === 'object'`  
   - **适用场景**：
     - 判断**基本类型**（除 `null`）  
     - 判断是否为函数（`typeof fn === 'function'`）  

2. **`instanceof` 的原理与特性**  
   - 表达式：`A instanceof B`  
   - 底层原理：判断 `B.prototype` 是否出现在 `A` 的原型链上  
     - 伪代码类似：

     ```js
     function myInstanceof(obj, Constructor) {
       let proto = Object.getPrototypeOf(obj);
       const target = Constructor.prototype;
       while (proto) {
         if (proto === target) return true;
         proto = Object.getPrototypeOf(proto);
       }
       return false;
     }
     ```

   - **适用场景**：
     - 判断某个对象是否为某个构造函数/类的实例，如 `arr instanceof Array`、`date instanceof Date`  
     - 适合在单一运行环境下（同一个 realm，如同一个窗口内）做对象类型判断  

3. **各自的局限性**  
   - `typeof` 的局限：
     - 无法区分 `null` 和普通对象：两者都显示 `'object'`（`null` 实际是特殊的基本类型）  
     - 对所有非函数对象（数组、正则、`Date` 等）都只返回 `'object'`，精细度不够  
   - `instanceof` 的局限：
     - 只能判断引用类型，**对基本类型不适用**：`1 instanceof Number` 为 `false`（字面量不是 `Number` 对象）  
     - 受原型链影响：如果原型链被改写，结果可能不可靠  
     - 不同 iframe / realm 下构造函数不共享原型链，`instanceof` 结果可能为 `false`（跨窗口问题）  

4. **综合使用建议**  
   - 粗粒度判断类型（特别是基本类型）：用 `typeof`  
   - 判断是否为数组：用 `Array.isArray`，而不是 `instanceof Array` 或 `typeof`  
   - 精细判断内置对象类型（`Array`、`Date`、`RegExp` 等）：用 `Object.prototype.toString.call(value)`  
   - 判断业务对象是否由某个类/构造函数创建：用 `instanceof`  

理解 `typeof` 和 `instanceof` 的实现原理，可以帮助我们在不同场景选用正确的类型检测方式，避免踩到 `typeof null` 等经典坑。
