## 🧩 题目

如何准确判断一个值是否为数组？有哪些常见方法？它们各自的优缺点是什么？

---

## 💡 解题思路

1. 列出常见方法：`Array.isArray`、`instanceof Array`、`Object.prototype.toString.call`
2. 分析跨 iframe/realm 情况下的可靠性
3. 对比实现复杂度、可读性，给出推荐方案

---

## ✅ 面试回答（标准版）

1. **`Array.isArray(value)`（首选）**  

```js
Array.isArray([]);        // true
Array.isArray({});        // false
Array.isArray('string');  // false
```

   - ES5.1 引入的标准方法，专门用来判断数组  
   - 能正确处理跨 iframe / 跨 realm 的数组，因为底层会检查内部 `[[Class]]` / `@@toStringTag` 等标记，而不是简单对比构造函数  
   - **推荐作为判断数组的首选方式**  

2. **`value instanceof Array`**  

```js
[] instanceof Array;      // true
({}) instanceof Array;    // false
```

   - 原理：判断 `Array.prototype` 是否出现在 `value` 的原型链上  
   - 在大多数单页面场景下是可行的，但**在跨 iframe / 跨窗口**时会失效：不同全局环境有各自的 `Array` 构造函数和原型链  
   - 如果 `Array.prototype` 被篡改，也可能导致结果不可靠  

3. **`Object.prototype.toString.call(value)`**  

```js
Object.prototype.toString.call([]); // "[object Array]"
```

   - 可以区分很多内置类型：`[object Array]`、`[object Object]`、`[object Date]` 等  
   - 写法相对啰嗦，阅读成本稍高，但在需要统一类型判断工具时很有用  

4. **不推荐的方法**  
   - `typeof [] === 'object'`：无法区分普通对象和数组  
   - `value.constructor === Array`：可能被修改或在无原型对象（`Object.create(null)`）等场景下出问题  

5. **实践建议**  
   - **简单项目/日常开发**：直接使用 `Array.isArray(value)`，语义清晰、可读性好  
   - **需要兼容非常老的环境（不支持 `Array.isArray`）**：可以用 `Object.prototype.toString.call(value)` 或给 `Array.isArray` 做一个 polyfill  

一句话：判断数组最推荐的方法是 `Array.isArray`，其次是 `Object.prototype.toString.call`，`instanceof Array` 只在单一运行环境里作为辅助。
