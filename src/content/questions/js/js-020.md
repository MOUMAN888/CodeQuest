## 🧩 题目

Promise 的核心原理是什么？如何手写实现 `Promise.all`、`Promise.race`、`Promise.allSettled` 这几个常用静态方法？它们之间的区别是什么？

---

## 💡 解题思路

1. 回顾 Promise 状态机：pending / fulfilled / rejected，不可逆
2. then 回调队列与微任务执行
3. 分别说明 `all`、`race`、`allSettled` 的语义与触发时机
4. 给出简化版伪实现，强调对 thenable 的处理和错误捕获

---

## ✅ 面试回答（标准版）

1. **Promise 的核心原理（简要版）**  
   - Promise 是一个**状态机对象**，有三种状态：
     - `pending`：进行中  
     - `fulfilled`：已成功  
     - `rejected`：已失败  
   - 状态一旦从 `pending` 变为 `fulfilled` 或 `rejected` 就**不可再改变**（不可逆）  
   - 通过 `then` / `catch` 注册的回调不会立即执行，而是被放入微任务队列，在当前宏任务结束后、渲染前执行  
   - 对于 `then` 返回的新 Promise，会根据回调的返回值（普通值 / Promise / thenable / 抛错）来决定新 Promise 的状态，这就是 Promise 链式调用的基础  

2. **`Promise.all` 的语义与实现思路**  
   - 接收一个可迭代对象（通常是数组），**等待所有 Promise 成功**，才整体成功；一旦有一个失败，就立即整体失败  
   - 结果成功时返回一个数组，对应每个输入 Promise 的成功结果，顺序与输入顺序一致  

   简化实现思路（伪代码）：

   ```js
   Promise.myAll = function (iterable) {
     return new Promise((resolve, reject) => {
       const arr = Array.from(iterable);
       const results = new Array(arr.length);
       let count = 0;

       if (arr.length === 0) return resolve([]);

       arr.forEach((p, index) => {
         Promise.resolve(p).then(
           value => {
             results[index] = value;
             if (++count === arr.length) {
               resolve(results);
             }
           },
           err => reject(err)
         );
       });
     });
   };
   ```

3. **`Promise.race` 的语义与实现思路**  
   - 接收一组 Promise，**哪个先有结果（成功或失败），就以该结果来 settle 整个 race**  

   ```js
   Promise.myRace = function (iterable) {
     return new Promise((resolve, reject) => {
       for (const p of iterable) {
         Promise.resolve(p).then(resolve, reject);
       }
     });
   };
   ```

4. **`Promise.allSettled` 的语义与实现思路**  
   - 等待所有 Promise 都结束（无论成功还是失败），**永远不会 reject**，而是得到每个任务的状态与对应值/原因  

   ```js
   Promise.myAllSettled = function (iterable) {
     return new Promise(resolve => {
       const arr = Array.from(iterable);
       const results = new Array(arr.length);
       let count = 0;

       if (arr.length === 0) return resolve([]);

       arr.forEach((p, index) => {
         Promise.resolve(p)
           .then(value => {
             results[index] = { status: 'fulfilled', value };
           })
           .catch(reason => {
             results[index] = { status: 'rejected', reason };
           })
           .finally(() => {
             if (++count === arr.length) {
               resolve(results);
             }
           });
       });
     });
   };
   ```

5. **三者的区别总结**  
   - `Promise.all`：**全成功才成功**，有一个失败就失败，适合「强一致」场景  
   - `Promise.race`：**谁先结束用谁的结果**，可用于超时控制、竞态请求等  
   - `Promise.allSettled`：**等全部结束，永不 reject**，适合「需要知道每个任务最终状态」的场景，如批量接口调用结果统计  

理解 Promise 的状态流转和这几个静态方法的语义，有助于在复杂异步场景下做出正确的抽象与 API 选择。
