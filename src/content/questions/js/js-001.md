## 🧩 题目

请解释 JavaScript Event Loop 的执行机制。

---

## 💡 解题思路

1. JS 是单线程，所有任务都在同一个主线程上执行
2. 同步任务直接在 Call Stack 中执行
3. 异步任务完成后会被放入 MacroTask / MicroTask 队列
4. 每一轮循环：先清空所有 microtask，再取一个 macrotask

---

## ✅ 面试回答（标准版）

Event Loop 是 JavaScript 在浏览器和 Node.js 中的事件循环机制，用来协调同步任务和异步任务的执行。JS 引擎只有一个主线程，同步任务会直接压入调用栈执行；异步任务（如 `setTimeout`、`Promise`）完成后，会把回调放入不同的任务队列。每一轮循环，先执行完当前调用栈中的同步任务，然后按顺序清空微任务队列（microtask，如 `Promise.then`），再取出一个宏任务（macrotask，如 `setTimeout` 回调）执行，如此往复，这就是事件循环。
