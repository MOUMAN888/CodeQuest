## 🧩 题目

JavaScript 可选链操作符（`?.`）的作用是什么？它是如何解决属性访问中的空值问题的？除此之外，还有哪些类似的语法糖（如 `??`），适用于哪些场景？

---

## 💡 解题思路

1. 可选链 `?.`：安全访问深层属性，避免 `Cannot read property ... of undefined`
2. 空值合并运算符 `??`：区分 `null/undefined` 与其他假值
3. 说明与 `||` 的差异，以及常见组合使用方式
4. 简要提及「可选链 + 可选调用」等多种形式

---

## ✅ 面试回答（标准版）

1. **可选链操作符 `?.` 的作用**  
   - 用于在访问对象深层属性或调用方法时，**自动进行空值（`null/undefined`）短路判断**，避免抛出运行时错误：  

   ```js
   const city = user?.address?.city;
   // 等价于：
   const city2 =
     user && user.address ? user.address.city : undefined;
   ```

   - 如果中间任意一层是 `null` 或 `undefined`，表达式直接返回 `undefined`，不会再继续向下访问，因此不会出现 `Cannot read property 'xxx' of undefined` 这类错误  

2. **可选链的几种常见形式**  
   - 属性访问：`obj?.prop` / `obj?.[propName]`  
   - 方法调用：`obj.method?.()`（避免方法不存在时的调用错误）  
   - 数组/链式调用中的组合：`arr?.[0]?.name`  

3. **空值合并运算符 `??` 的作用与区别**  
   - `a ?? b` 的语义是：**如果 `a` 为 `null` 或 `undefined`，则返回 `b`，否则返回 `a` 本身**  
   - 与 `||` 的区别在于：`||` 把所有「假值」（`0`、`''`、`false`、`NaN`、`null`、`undefined`）都视为「需要回退默认值」，而 `??` 只把 `null/undefined` 当成「空值」：  

   ```js
   0 || 42;       // 42
   0 ?? 42;       // 0
   '' || 'def';   // 'def'
   '' ?? 'def';   // ''
   null ?? 'def'; // 'def'
   ```

   - 因此在配置项、数值为 0、空字符串本身合法等场景下，**更推荐使用 `??`** 来设置默认值  

4. **组合使用示例**  

```js
const city = user?.address?.city ?? '未知城市';
// 含义：如果 user 或 address 为空，则 city 为 '未知城市'，
// 否则为实际的 city 值（即使是空字符串也保留）
```

5. **实践建议**  
   - 在访问深层嵌套对象、接口返回数据结构不稳定、可选字段很多时，使用 `?.` 可以显著减少重复的空值判断代码  
   - 在设置默认值时，明确区分「空字符串、0、false 是合法值」和「缺失值（null/undefined）」的语义，选择 `??` 而不是 `||`  

可选链和空值合并运算符本质上都是帮助我们更**安全、明确地处理空值**的语法糖，是现代 JavaScript 推荐使用的模式。
