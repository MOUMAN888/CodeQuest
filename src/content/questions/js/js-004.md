## 🧩 题目

JavaScript 有哪些数据类型？基本类型和引用类型的区别是什么？常见的类型判断方式有哪些？适用在什么场景？

---

## 💡 解题思路

1. 列出所有数据类型：7 个基本类型 + 1 个引用类型（Object 家族）
2. 区分基本类型与引用类型：存储位置、赋值 & 传参时的拷贝方式
3. 常见类型判断方式：`typeof`、`instanceof`、`Object.prototype.toString`、`Array.isArray` 等
4. 各方式的优缺点与典型使用场景

---

## ✅ 面试回答（标准版）

1. **JavaScript 中的数据类型**  
   按照 ES 规范，JS 中有 7 种**基本类型**（Primitive）和 1 大类**引用类型**：
   - 基本类型：
     - `number`
     - `string`
     - `boolean`
     - `null`
     - `undefined`
     - `symbol`
     - `bigint`
   - 引用类型：
     - `object`（包括普通对象、数组、函数、Date、RegExp、Map、Set 等等）

2. **基本类型 vs 引用类型的区别**  
   - **存储方式**：
     - 基本类型的值直接存放在**栈内存**（或等价的连续区域）中，访问时是按值访问  
     - 引用类型的变量在栈中存放的是一个**引用（指针）**，真正的数据存放在堆内存中  
   - **赋值 & 传参行为**：
     - 基本类型赋值是「值拷贝」：修改新变量不会影响原变量  
     - 引用类型赋值是「引用拷贝」：多个变量指向同一块堆内存，修改其中一方会影响其他引用  
   - **不可变性**：
     - 基本类型本身不可变（重新赋值相当于创建了新值）  
     - 引用类型内部结构可变（可以在不改变引用本身的情况下修改其属性）  

3. **常见的类型判断方式**  
   - **`typeof`**  
     - 适合区分：`number` / `string` / `boolean` / `undefined` / `symbol` / `bigint` / `function`  
     - 特别注意：`typeof null === 'object'` 是历史遗留问题，不能用 `typeof` 区分 `null` 和普通对象  
   - **`instanceof`**  
     - 用于判断**引用类型的原型链**关系：`obj instanceof Constructor`  
     - 原理：判断 `Constructor.prototype` 是否出现在 `obj` 的原型链上  
     - 适合判断：自定义类实例、`Date`、`RegExp` 等，但不适用于基本类型（`1 instanceof Number` 为 `false`）  
   - **`Object.prototype.toString.call(value)`**  
     - 返回类似 `[object Array]`、`[object Date]` 的字符串，是一种更精确的类型检测方式  
     - 适合区分数组、对象、Date、RegExp、`null`、`undefined` 等：  
       - `Object.prototype.toString.call([]) === '[object Array]'`  
       - `Object.prototype.toString.call(null) === '[object Null]'`  
   - **`Array.isArray(value)`**  
     - 专门用来判断数组，比 `obj instanceof Array` 更安全（跨 iframe/realm 时也可靠）  
   - **`value.constructor` / `value.__proto__`**（不推荐在通用库中依赖）  
     - 会受原型链修改、`Object.create(null)` 等情况影响，有一定局限  

4. **实战中的选择建议**  
   - 判断是否为 `null` / `undefined`：用严格等号 `=== null` / `=== undefined` 或 `value == null`（同时判断两者）  
   - 判断基本类型：首选 `typeof`  
   - 判断是否为数组：`Array.isArray(value)`  
   - 判断更细的内置对象类型：`Object.prototype.toString.call(value)`  
   - 判断实例是否来自某个构造函数/类：`instanceof`，用于业务对象和类实例判断  

整体来说，**没有一种万能的类型判断方式**，要根据具体需求选择合适的方法，理解底层原理比死记 API 更重要。
