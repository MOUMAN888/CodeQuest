## 🧩 题目

JavaScript 中 `for...in` 和 `for...of` 有什么区别？它们分别适用于哪些场景？使用时有哪些注意事项？

---

## 💡 解题思路

1. `for...in`：遍历**可枚举属性的键名**（包括原型链上可枚举属性）
2. `for...of`：遍历**可迭代对象的值**（基于 `Symbol.iterator`）
3. 各自适用：对象遍历 vs 数组/字符串/Map/Set 等遍历
4. 注意点：性能、顺序、是否遍历原型链、与 `Object.keys` 搭配使用等

---

## ✅ 面试回答（标准版）

1. **`for...in` 的特点与用途**  
   - 用于遍历对象**可枚举属性的键名**（key），包括其原型链上的可枚举属性  
   - 遍历的是字符串键，如 `'0'、'1'、'name'`：  

   ```js
   const obj = { a: 1, b: 2 };
   for (const key in obj) {
     console.log(key, obj[key]);
   }
   ```

   - 对数组使用 `for...in` 时，得到的是索引字符串，而且还可能枚举出手动添加的属性以及原型链上的属性，因此**不推荐用于遍历数组**  

2. **`for...of` 的特点与用途**  
   - 用于遍历**可迭代对象（Iterable）** 的**值**（value），本质上依赖对象的 `Symbol.iterator` 接口  
   - 常见可迭代对象包括：数组、字符串、`Map`、`Set`、`arguments`、TypedArray 等  

   ```js
   const arr = [10, 20, 30];
   for (const value of arr) {
     console.log(value);
   }
   ```

   - 对于 `Map` 和 `Set`：  
     - `for (const [key, value] of map)`  
     - `for (const value of set)`  

3. **对比总结**  
   - **遍历内容不同**：
     - `for...in`：遍历键名（key），且会遍历原型链上的可枚举属性  
     - `for...of`：遍历值（value），不关心键名，基于迭代器协议  
   - **适用对象不同**：
     - `for...in`：更适合一般对象的属性遍历（也建议配合 `hasOwnProperty` 过滤继承属性）  
     - `for...of`：更适合数组、字符串、`Map`、`Set` 等**有顺序的可迭代集合**  
   - **顺序与性能**：
     - `for...in` 的遍历顺序不保证与插入顺序一致，且性能相对较差  
     - `for...of` 遵循迭代器定义的顺序，对数组等集合更合理  

4. **实践建议**  
   - 遍历数组或类数组：**优先使用 `for` / `forEach` / `for...of`**，避免 `for...in`  
   - 遍历对象属性：  
     - 使用 `Object.keys` / `Object.entries` 搭配 `for...of`：  

       ```js
       for (const [key, value] of Object.entries(obj)) { ... }
       ```

     - 或 `for...in` 搭配 `hasOwnProperty` 使用：  

       ```js
       for (const key in obj) {
         if (obj.hasOwnProperty(key)) { ... }
       }
       ```

一句话总结：**`for...in` 遍历「键」，更偏向对象；`for...of` 遍历「值」，更偏向集合/序列。**
