## 🧩 题目

JavaScript 事件循环（Event Loop）的执行机制是什么？宏任务（MacroTask）和微任务（MicroTask）分别包含哪些？请结合一段代码分析具体的执行顺序。

---

## 💡 解题思路

1. 回顾 JS 单线程模型：调用栈 + 任务队列
2. 宏任务 vs 微任务：常见 API 分类
3. 每一轮 Event Loop 的步骤：执行栈 → 清空微任务 → 渲染 → 下一个宏任务
4. 给一段带有 `setTimeout`、`Promise.then` 的代码，分析输出顺序

---

## ✅ 面试回答（标准版）

1. **事件循环的基本模型**  
   - JS 是单线程的，所有代码都在同一个主线程上执行  
   - 同步任务直接进入调用栈（Call Stack）按顺序执行  
   - 异步任务完成后会将回调放入不同的任务队列中，等主线程空闲时再根据事件循环机制调度执行  

2. **宏任务（MacroTask）与微任务（MicroTask）**  
   - 常见的 **宏任务** 包括：  
     - `setTimeout`、`setInterval`  
     - `setImmediate`（Node）  
     - I/O 回调、UI 渲染等  
   - 常见的 **微任务** 包括：  
     - `Promise.then` / `catch` / `finally` 回调  
     - `queueMicrotask`  
     - `MutationObserver` 回调  

3. **每一轮 Event Loop 的关键步骤（简化模型）**  
   1）执行当前宏任务中的所有同步代码，直到调用栈清空  
   2）**依次取出微任务队列中的所有任务并执行，直到微任务队列被清空**  
   3）如果需要，进行一次 UI 渲染  
   4）从宏任务队列中取出下一个宏任务，重复上述过程  

4. **示例代码分析执行顺序**  

```js
console.log('script start');

setTimeout(() => {
  console.log('setTimeout');
}, 0);

Promise.resolve()
  .then(() => {
    console.log('promise1');
  })
  .then(() => {
    console.log('promise2');
  });

console.log('script end');
```

执行顺序分析：  
1）先执行所有同步代码：  
   - 输出：`script start`  
   - 注册 `setTimeout` 回调到宏任务队列  
   - 注册 `Promise.then` 回调到微任务队列  
   - 输出：`script end`  
2）当前宏任务中的同步代码执行完毕，开始处理**微任务队列**：  
   - 执行第一个 `then`：输出 `promise1`，并将下一个 `then` 回调加入微任务队列  
   - 继续执行微任务：输出 `promise2`  
3）微任务全部执行完毕，进入下一轮事件循环，从宏任务队列取出 `setTimeout` 回调执行：  
   - 输出：`setTimeout`  

最终输出顺序为：

```text
script start
script end
promise1
promise2
setTimeout
```

理解事件循环和宏/微任务的执行顺序，是正确编写和调试异步代码（尤其是 Promise 和定时器混用场景）的基础。
