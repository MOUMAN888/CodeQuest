## 🧩 题目

什么是块级作用域、全局作用域、函数作用域？`let`、`const`、`var` 之间有什么区别？在实际开发中应该如何选择？

---

## 💡 解题思路

1. 定义三种作用域：全局、函数、块级（`{}`、`if`、`for` 等）
2. `var`：函数作用域、变量提升、可重复声明、挂载到全局对象
3. `let`/`const`：块级作用域、暂时性死区、不可重复声明
4. `const`：绑定不可变 vs 引用内容可变
5. 实践推荐：默认 `const`，需要重新赋值时用 `let`，尽量避免 `var`

---

## ✅ 面试回答（标准版）

1. **三种作用域的概念**  
   - **全局作用域**：  
     - 在任何函数或块外声明的变量处于全局作用域，在整个程序运行期间都可访问  
     - 在浏览器中，使用 `var` 在全局声明的变量会被挂到 `window` 对象上  
   - **函数作用域**：  
     - 每个函数调用都会创建一个独立的作用域，函数内部声明的变量默认只在该函数内可见  
   - **块级作用域**：  
     - ES6 引入，任意一对花括号 `{}`（如 `if`、`for`、`while`、普通代码块）都可以形成一个新的作用域  
     - 使用 `let` / `const` 声明的变量会被限制在当前块级作用域内  

2. **`var` 的特点**  
   - **函数作用域**：不支持块级作用域，`if`、`for` 等语句块不会限制 `var` 变量的可见性  
   - **变量提升（Hoisting）**：声明会被提升到当前作用域顶部，初始化仍在原位置 → 容易产生 `undefined` 和逻辑错误  
   - **可重复声明**：同一作用域内可以多次 `var` 同名变量，不会抛错  
   - 在全局使用 `var` 声明，会在浏览器环境中把变量挂到 `window` 上，容易造成全局污染  

3. **`let` 的特点**  
   - **块级作用域**：只在当前 `{}` 中有效，常用于 `for` 循环计数器等场景，避免变量泄露到外层  
   - **不存在变量提升的可见性问题**：虽然从规范角度仍然会有声明提升，但在「声明之前访问」会触发 **暂时性死区（TDZ）** 抛错  
   - **不可重复声明**：同一作用域下重复声明会报错，减少命名冲突  
   - 适合用来声明会被重新赋值的普通局部变量  

4. **`const` 的特点**  
   - 与 `let` 一样是**块级作用域**，同样存在 TDZ 和不可重复声明  
   - 区别在于：`const` 声明的是**只读绑定**，必须在声明时初始化，之后不能再重新赋值  
   - 但如果绑定的是一个对象/数组，**对象本身的内容依然可以修改**：

   ```js
   const obj = { a: 1 };
   obj.a = 2;  // ✅ 允许
   obj = {};   // ❌ 不允许
   ```

5. **实践中的选择建议**  
   - 通常推荐：**默认使用 `const`，只有确定需要重新赋值时才用 `let`**  
   - `var` 仅在需要兼容非常老的环境或特定场景（如故意利用函数提升）时使用，一般项目中应尽量避免  
   - 合理利用块级作用域与 `let`/`const` 可以避免很多由变量提升、全局污染引起的 Bug  

一句话总结：`var` 基于函数作用域且有变量提升，容易出坑；`let`/`const` 引入了块级作用域和暂时性死区，是现代 JavaScript 中推荐使用的变量声明方式。
