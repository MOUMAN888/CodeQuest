## 🧩 题目

请详细讲解 JavaScript 原型链的原理、原型继承的常见实现方式，以及在实际开发中的应用场景。

---

## 💡 解题思路

1. 回顾：对象、构造函数、`prototype`、`[[Prototype]]`（`__proto__`）
2. 原型链查找规则，直到 `Object.prototype` 与 `null`
3. 继承实现方式：构造函数继承、原型链继承、组合继承、寄生组合继承、`class extends`
4. 常见应用：方法复用、多实例共享方法、自定义类型判断等

---

## ✅ 面试回答（标准版）

1. **原型与原型链的核心概念**  
   - 每个对象在创建时都会关联一个内部属性 `[[Prototype]]`，在大多数引擎中可以通过非标准属性 `__proto__` 访问  
   - 构造函数的 `prototype` 属性指向其实例对象的原型：

     ```js
     function Person() {}
     const p = new Person();
     console.log(p.__proto__ === Person.prototype); // true
     ```

   - 当访问对象的某个属性时，如果对象自身没有，就会沿着 `[[Prototype]]` 一层一层向上查找，这条由原型对象串起来的链路就叫**原型链**  
   - 顶层是 `Object.prototype`，再往上是 `null`，表示原型链的尽头  

2. **原型链的查找过程**  
   - 假设访问 `obj.someProp`：  
     1）先在 `obj` 自身的属性上查找  
     2）如果没有，就到 `obj.__proto__`（即其原型）上查找  
     3）再没有，就继续到原型的原型，直到 `Object.prototype`  
     4）如果整个链上都没有，返回 `undefined`  
   - 方法调用时也是如此：`obj.toString()` 实际调用的是 `Object.prototype.toString`  

3. **常见的原型继承实现方式**  
   - **原型链继承**（只改 `Child.prototype`）：

     ```js
     function Parent() { this.name = 'parent'; }
     Parent.prototype.say = function () {};

     function Child() {}
     Child.prototype = new Parent();
     Child.prototype.constructor = Child;
     ```

     缺点：所有子实例共享父实例的引用属性，容易互相干扰  
   - **构造函数继承**（只在子构造函数中调用父构造函数）：

     ```js
     function Child() {
       Parent.call(this); // 只继承实例属性
     }
     ```

     缺点：方法无法复用，每个实例都有一份  
   - **组合继承**（构造函数继承 + 原型链继承）：

     ```js
     function Child() {
       Parent.call(this); // 继承实例属性
     }
     Child.prototype = new Parent(); // 继承原型方法
     Child.prototype.constructor = Child;
     ```

     缺点：`Parent` 被调用了两次（一次给实例属性，一次给原型）  
   - **寄生组合继承**（推荐的 ES5 写法）：

     ```js
     function inherit(child, parent) {
       child.prototype = Object.create(parent.prototype);
       child.prototype.constructor = child;
     }
     ```

     - 用 `Object.create` 以父原型为原型创建一个中间对象，避免多次调用父构造函数  
   - **ES6 `class` 语法糖**：

     ```js
     class Parent {
       say() {}
     }

     class Child extends Parent {
       constructor() {
         super();
       }
     }
     ```

     本质上仍然是基于原型链的继承，只是语法更易读  

4. **常见应用场景**  
   - 定义「类」和「实例方法」，通过原型共享方法，节省内存  
   - 实现多层继承结构，如 `EventEmitter`、错误类型（`Error`、`TypeError` 等）  
   - 配合 `instanceof` 做类型判断：检查构造函数的 `prototype` 是否在实例的原型链上  

总结：原型链是 JavaScript 对象系统的基础，所有继承和方法复用最终都落在「**对象通过原型链查找属性**」这一条规则上。ES6 的 `class` 只是语法糖，理解底层的原型链有助于更好地调试和设计对象结构。
