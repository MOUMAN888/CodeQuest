## 🧩 题目

`instanceof` 运算符是基于什么原理来判断类型的？为什么它不能准确判断基本类型？在什么场景下适合使用 `instanceof`？

---

## 💡 解题思路

1. 给出原理：判断右侧构造函数的 `prototype` 是否出现在左侧对象的原型链上
2. 说明仅适用于引用类型实例，基本类型字面量不是构造出来的对象
3. 说明包装对象（如 `new Number(1)`）的特例
4. 总结适用场景与替代方案

---

## ✅ 面试回答（标准版）

1. **`instanceof` 的判断原理**  

表达式：`A instanceof B`  

其判断逻辑是：**检查构造函数 `B` 的 `prototype` 属性是否出现在对象 `A` 的原型链上**。  

伪代码大致如下：

```js
function myInstanceof(obj, Constructor) {
  if (obj == null) return false;
  let proto = Object.getPrototypeOf(obj);
  const target = Constructor.prototype;
  while (proto) {
    if (proto === target) return true;
    proto = Object.getPrototypeOf(proto);
  }
  return false;
}
```

只要在沿着 `obj.__proto__`、`obj.__proto__.__proto__`... 的查找过程中能遇到 `Constructor.prototype`，结果就是 `true`。  

2. **为什么不能准确判断基本类型？**  
   - 基本类型（`number`、`string`、`boolean`、`symbol`、`bigint`、`null`、`undefined`）不是通过构造函数创建的对象，没有原型链可言：  

   ```js
   1 instanceof Number;       // false
   'a' instanceof String;     // false
   true instanceof Boolean;   // false
   ```

   - 上面的 `Number`、`String`、`Boolean` 是**包装对象构造函数**，而字面量是基本类型  
   - 只有使用 `new Number(1)` 等创建出来的包装对象才会返回 `true`：  

   ```js
   new Number(1) instanceof Number; // true
   ```

   - 因此，`instanceof` 无法直接用于判断一个值是否为「基本类型」，它只适用于**对象实例与构造函数/类**之间的关系判断  

3. **适用场景**  
   - 判断某个对象是否为某个构造函数或类的实例：  

   ```js
   obj instanceof MyClass;
   date instanceof Date;
   error instanceof Error;
   ```

   - 在框架/库内部，用于根据实例类型分发逻辑（如区分不同的错误类型、自定义类层级等）  

4. **局限性与替代方案**  
   - 跨 iframe / 跨 realm 时，`instanceof` 可能失效，因为不同全局环境有独立的构造函数与原型链  
   - 原型链被修改或打断时，`instanceof` 结果也可能和预期不符  
   - 对于基本类型或需要跨环境判断时，更适合使用：
     - `typeof` 来判断基本类型  
     - `Object.prototype.toString.call(value)` 区分内置类型  

一句话：`instanceof` 是基于原型链的「构造函数-实例」关系检查工具，只适合用于引用类型实例判断，而不能用来准确判断基本类型。
