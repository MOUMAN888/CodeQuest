## 🧩 题目

数组 `[]` 存储在堆内存还是栈内存？JavaScript 的内存模型大致是怎样的？请说明基本类型与引用类型在内存中的存储方式和核心规则。

---

## 💡 解题思路

1. 先回答：数组是对象，属于引用类型，其内容在堆上，变量中存的是引用
2. 简要说明 JS 的「栈 + 堆」模型抽象（尽管不同引擎实现细节不同）
3. 对比基本类型和引用类型的存储与传递方式
4. 说明垃圾回收的基本思想：可达性、引用计数/标记清除等（简略）

---

## ✅ 面试回答（标准版）

1. **数组存储在堆还是栈？**  
   - 在 JavaScript 中，数组本质上是一种特殊的对象（`typeof [] === 'object'`），属于**引用类型**  
   - 一般的抽象是：  
     - **变量本身**（如 `const arr = ...`）存放在栈中，里面保存了一个**引用（指针）**  
     - 数组的真实数据（元素、内部结构）存放在**堆内存**中  
   - 因此，「`[]` 存储在堆还是栈」更准确的说法是：**数组对象本身在堆上，变量在栈上存储对它的引用**  

2. **JavaScript 内存模型的核心抽象**  
   尽管不同 JS 引擎实现存在差异，我们通常用「栈 + 堆」来理解：  
   - **栈（Stack）**：  
     - 存放执行上下文（函数调用栈）、局部变量的引用等  
     - 访问速度快，内存空间相对较小  
   - **堆（Heap）**：  
     - 存放对象、数组、函数等引用类型的数据  
     - 空间大，布局灵活，由垃圾回收器管理生命周期  

3. **基本类型 vs 引用类型的内存行为**  
   - **基本类型**（number、string、boolean、null、undefined、symbol、bigint）：  
     - 值本身通常直接存储在栈中（有的引擎可能会做逃逸分析优化，这里按经典抽象理解）  
     - 赋值或传参时是**值拷贝**：改动新变量不会影响原变量  
   - **引用类型**（Object、Array、Function 等）：  
     - 变量在栈上存的是**指向堆中对象的引用**  
     - 赋值或传参时是**引用拷贝**：多个变量指向同一块堆内存，对其属性的修改会相互可见  

4. **垃圾回收的基本思想（简要）**  
   - JS 引擎会自动回收不再使用的内存，开发者无法直接控制释放  
   - 常见思想是基于「可达性（Reachability）」：  
     - 从根对象（如全局对象、当前调用栈中的变量）出发，能通过引用链到达的对象视为「存活」  
     - 不可达的对象会被视为垃圾，在某个时间点统一回收（如标记-清除算法）  
   - 因此，只要还有变量指向某个对象，该对象就不会被释放；一旦所有引用都被断开（如设为 `null`），在后续回收周期中就会被清理  

5. **实践中的启示**  
   - 了解「数组/对象在堆上，变量存引用」有助于正确理解浅拷贝/深拷贝、函数传参、副作用等问题  
   - 避免不必要地在长生命周期对象上挂载大对象/闭包引用，以免延迟释放内存  

总结：数组和对象一样，都是存储在堆上的引用类型；变量本身保存的是指向这些堆对象的引用，这构成了 JavaScript 内存模型中理解「值」与「引用」的基础。
