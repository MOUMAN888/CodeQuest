## 🧩 题目

为什么不直接用原生 JavaScript 来开发大型前端项目？以 Vue/React 为代表的前端框架相比原生开发有哪些核心优势？

---

## 💡 解题思路

1. 先承认原生 JS 能实现一切，但在复杂度和可维护性上存在问题
2. 列出框架核心价值：组件化、状态管理、声明式 UI、生态与工程化支持
3. 举例说明：DOM 操作复杂性、多人协作、可测试性

---

## ✅ 面试回答（标准版）

1. **原生 JS 理论上能实现一切，但工程成本极高**  
   - 在小项目或简单页面中，用原生 JS + DOM API 完全可行。  
   - 但在大型项目中，需要处理：复杂的 UI 状态、路由、多模块协作、频繁交互与更新，如果完全依赖手写 DOM 操作和事件绑定，代码会迅速变得难以维护和扩展。  

2. **前端框架的核心优势**  
   - **声明式 UI 与组件化**：  
     - Vue/React 提供「**状态驱动视图**」的声明式模型（`state -> UI`），开发者关注「组件在某个状态下长什么样」，而不用手动追踪每个状态变化对应的 DOM 更新步骤。  
     - 通过组件化拆分复杂页面，形成可复用、可组合的 UI 单元，便于多人协作与代码重用。  
   - **统一的状态管理与数据流**：  
     - 提供类似 Vuex/Pinia、Redux/Zustand 等状态管理方案，使数据流更可控、可预测，方便调试与回溯。  
   - **生态与工程化支持**：  
     - 丰富的路由、表单、UI 组件库、状态管理、国际化、测试、构建工具等生态，减少造轮子时间。  
     - 与 TypeScript、Lint、测试框架等深度集成，有利于构建规范统一、质量可控的代码库。  
   - **跨平台与可移植性**：  
     - 同一套组件/逻辑可拓展到 Web、小程序、Native（如 React Native、Uni-app 等），提升代码复用度。  

3. **对团队协作与长期维护的价值**  
   - 框架提供了一套约定俗成的「组织代码的方式」，让团队成员更易读懂彼此代码。  
   - 有利于引入自动化测试（组件测试、快照测试）、性能优化工具、错误边界等工程能力。  
   - 当项目迭代多年，框架带来的结构化设计可以显著降低维护与重构成本。  

4. **总结**  
   - 原生 JS 更像是「底层语言」，适合在局部、无框架依赖的轻量场景使用；  
   - Vue/React 等框架则为复杂前端应用提供了「**结构化、可维护、可扩展**」的基础设施，是大型项目下「减少复杂度和人肉管理」的关键。
