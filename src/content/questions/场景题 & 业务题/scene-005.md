## 🧩 题目

请详细介绍你在实习期间负责的一个前端项目，包括项目背景、你的具体职责与贡献、最终成果以及项目中的核心亮点。

---

## 💡 解题思路

1. 结构化讲述：背景 → 目标 → 职责 → 难点/亮点 → 结果与数据
2. 强调你承担的角色：模块所有权、技术决策、协作沟通等
3. 突出 1–2 个技术或产品上的亮点：性能优化、工程化、体验提升、自动化等

---

## ✅ 面试回答（标准版）

在回答这类开放问题时，可以采用 STAR 结构（Situation、Task、Action、Result）来组织语言。下面给出一个通用模板示例，你可以替换为自己的真实经历：

1. **项目背景（Situation）**  
   - 「在 XX 公司实习期间，我主要参与了一个用于 **（比如：内部运营管理 / 数据看板 / 在线教育平台）** 的前端项目，主要技术栈是 Vue 3 + TypeScript + Vite（或 React + TS + Webpack）。  
   - 项目的核心目标是 **（例如：提升运营效率、减少人工操作、对接多个后端服务）**，预期由原来线下/Excel 操作迁移到统一的 Web 管理平台。」  

2. **个人职责与任务（Task）**  
   - 负责的模块或功能：  
     - 例如：「我主要负责了用户管理模块和部分数据可视化页面的开发」，包括页面搭建、接口对接、权限控制与表格筛选等。  
   - 在团队中的角色：  
     - 「除了日常开发，我还参与了部分工程化改进，例如引入 ESLint + Prettier 统一代码风格，搭建基础组件库等。」  

3. **关键行动与技术亮点（Action）**  
   可以选 1–2 个你做过的实质贡献展开说明：  
   - **性能/体验优化**：  
     - 「针对列表页面数据量较大导致加载卡顿的问题，我引入了分页与筛选接口优化，并在前端实现了请求防抖与虚拟滚动，减少一次性渲染的 DOM 数量。」  
   - **工程化与代码质量**：  
     - 「在项目中补充了 ESLint/Prettier/Husky 提交前检查流程，减少了代码风格不一致和低级错误的发生。」  
   - **可复用组件或工具的沉淀**：  
     - 「将多个页面重复使用的表格查询条件封装成一个可配置的 Filter 组件，后续多个模块可以通过简单配置快速接入，减少了重复代码。」  

4. **结果与数据（Result）**  
   - 用尽量具体的数据或反馈来说明效果：  
     - 「最终项目按期上线，日均处理订单量提升了 X%，运营同事反馈操作步骤从原来的 N 步减少到 M 步。」  
     - 「虚拟列表优化后，长列表渲染时间从 2s 降到 300ms 左右，滚动帧率更稳定。」  
     - 「通过引入自动化检查，代码 Review 中因格式问题的评论明显减少，提效明显。」  

5. **复盘与收获**  
   - 简要总结你在项目中学到的东西：  
     - 如「如何在业务需求与工程质量之间平衡」、「如何与后端/产品/设计高效协作」、「如何在现有代码上做增量优化而不破坏稳定性」等。  

一句话：把项目讲清楚的关键，是用「结构化 + 数据支撑」的方式突出你的作用和项目亮点，让面试官看到你不仅会写代码，还能在真实业务环境中解决问题、推动改进。
