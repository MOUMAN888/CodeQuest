## 🧩 题目

Vue 的插槽有哪些类型？作用域插槽（Scoped Slot）的使用场景是什么？它的实现原理大致是怎样的？

---

## 💡 解题思路

1. 列出插槽类型：默认插槽、具名插槽、作用域插槽（Vue3 中统一为函数插槽）
2. 使用场景：父组件决定展示结构，子组件提供部分数据/方法给插槽内容使用
3. 实现原理：本质是「以函数形式传入的插槽」，调用时传入 slot props
4. 补充 Vue2 `slot-scope` / `scopedSlots` 与 Vue3 `v-slot` / 函数组件的统一

---

## ✅ 面试回答（标准版）

1. **Vue 中的插槽类型**  
   - **默认插槽（Default Slot）**：不带名称的插槽  

     ```vue
     <!-- 子组件 -->
     <slot></slot>

     <!-- 父组件 -->
     <MyCard>一些默认内容</MyCard>
     ```

   - **具名插槽（Named Slot）**：通过 `name` 区分多个插槽区域  

     ```vue
     <!-- 子组件 -->
     <slot name="header"></slot>
     <slot></slot>
     <slot name="footer"></slot>

     <!-- 父组件 -->
     <MyCard>
       <template #header>标题</template>
       主体内容
       <template #footer>底部</template>
     </MyCard>
     ```

   - **作用域插槽（Scoped Slot）**：子组件向插槽内容暴露一部分「插槽 props」，由父组件使用这些数据来决定如何渲染  

2. **作用域插槽的使用场景**  
   - 当组件内部有一部分数据或状态需要交给父组件来自由渲染时，适合使用作用域插槽，例如：  
     - 表格组件 `Table` 提供每行的数据 `row`，父组件用插槽自定义单元格内容  
     - 列表、下拉选择、卡片等组件想把「渲染内容的控制权」交给调用方  

   示例：

   ```vue
   <!-- 子组件：List.vue -->
   <ul>
     <li v-for="item in items" :key="item.id">
       <slot :item="item">{{ item.text }}</slot>
     </li>
   </ul>

   <!-- 父组件 -->
   <List :items="list">
     <template #default="{ item }">
       <strong>{{ item.id }}</strong> - {{ item.text }}
     </template>
   </List>
   ```

3. **实现原理（简化理解）**  
   - 在运行时，插槽本质上是以**函数**的形式从父组件传递给子组件的：  
     - 父组件在编译模板时，会把插槽内容编译成一个接收「插槽 props」参数的渲染函数  
     - 子组件内部渲染时调用这个函数，并把需要暴露给插槽的 `item` 等数据作为参数传进来  
   - 可以理解为：  

     ```js
     // 父组件传入：
     slots.default = (slotProps) => /* 返回 vnode 树 */;

     // 子组件内部：
     const vnode = slots.default({ item });
     ```

   - Vue2 中这部分信息会通过 `scopedSlots` 传递，Vue3 中统一为 `slots` 上的函数接口  

4. **总结**  
   - 默认插槽 & 具名插槽解决的是「父组件把内容插到子组件预留位置」的问题  
   - 作用域插槽进一步解决了「子组件有数据，父组件来决定怎么用」的问题，本质是「带参数的插槽函数」，使组件在封装逻辑的同时，又能保持高度的 UI 灵活性。
