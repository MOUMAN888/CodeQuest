## 🧩 题目

简要对比 Vue 3 中组合式 API（Composition API）与选项式 API（Options API）的主要区别，以及在什么场景下更推荐使用组合式 API。

---

## 💡 解题思路

1. Options API：按 `data`、`methods`、`computed` 等选项组织代码
2. Composition API：按「功能逻辑」组织代码，使用 `setup`、`ref`、`reactive`、`computed` 等
3. 组合式 API 更利于逻辑抽离与复用（`useXxx` 函数）
4. 大型组件 / 复杂业务逻辑更适合用组合式 API

---

## ✅ 面试回答（标准版）

在 Options API 中，我们会把数据、方法、计算属性、生命周期等分别写在 `data`、`methods`、`computed`、`mounted` 等不同的配置项里，代码是按「类型」分组的；而在 Composition API 中，我们使用 `setup` 函数，通过 `ref`、`reactive`、`computed`、`watch` 等组合出一块块逻辑，代码是按「功能」分组的。  
当组件逻辑比较简单时，两者都能写得很清晰；但在复杂组件中，Options API 往往会出现「同一块业务逻辑散落在多个选项里」的问题，而 Composition API 可以把同一业务逻辑放在一起，甚至抽成独立的 `useXxx` 组合函数在多个组件间复用。因此，在**中大型项目、复杂业务逻辑、需要逻辑复用**的场景下，更推荐使用组合式 API。
