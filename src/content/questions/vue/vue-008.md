## 🧩 题目

在 Vue 3 中，父子组件的 `setup`、`render`、`mounted` 等生命周期钩子大致的执行先后顺序是什么？请分别说明「挂载」和「卸载」两个阶段的调用顺序。

---

## 💡 解题思路

1. 分阶段说明：初次挂载、更新、卸载
2. 挂载阶段：先父后子（setup/render），再子后父（mounted）
3. 卸载阶段：先父后子触发卸载，再子后父执行 `unmounted`
4. 简要附带 `beforeMount`、`beforeUpdate`、`updated` 的顺序，突出记忆主干

---

## ✅ 面试回答（标准版）

可以按「**先父后子挂载，先子后父完成挂载**」来记住大致顺序；卸载时则是「**先父触发卸载，再子后父执行真正的销毁钩子**」。

1. **初次挂载阶段（父 P，子 C）**  
   在 Vue 3 中，组合式 API 的 `setup` 会在选项式 API 钩子前执行，大致顺序可简化为：
   - 创建组件实例 & 初始化：  
     1）父组件：`setup(P)` → 生成渲染函数 / 返回上下文  
     2）父组件：执行 `render(P)` 过程中遇到子组件 → 创建子组件实例  
     3）子组件：`setup(C)`  
     4）子组件：`render(C)`  
   - 挂载到 DOM 时的生命周期钩子（不考虑 keep-alive 等特殊情况）：  
     - 先挂载子，再挂载父，顺序类似：  
       - 子：`onBeforeMount(C)` → 挂载子 DOM → `onMounted(C)`  
       - 父：`onBeforeMount(P)` → 挂载父 DOM → `onMounted(P)`  

   用更口语化的顺序记忆：  
   - **挂载前 & 渲染准备：父 setup → 父 render 遇到子 → 子 setup → 子 render**  
   - **DOM 挂载完成：先子 mounted，后父 mounted**  

2. **更新阶段（父更新引起子更新的情况）**  
   - 数据变化触发重新渲染时，大致顺序（简化）：
     1）父组件响应式数据变化 → 触发父的 `render`，在过程中更新子组件的 props  
     2）子组件根据新的 props 触发自身更新  
   - 生命周期钩子顺序可以简化记忆为：  
     - 更新前：**父 `beforeUpdate` → 子 `beforeUpdate`**  
     - 视图更新 & patch  
     - 更新后：**子 `updated` → 父 `updated`**  

3. **卸载阶段（销毁组件）**  
   - 卸载时的顺序可概括为「父先触发卸载，子先真正销毁」：  
     - 触发阶段：  
       1）父组件：`onBeforeUnmount(P)`  
       2）子组件：`onBeforeUnmount(C)`  
     - 真正执行卸载与清理：  
       3）子组件：`onUnmounted(C)`  
       4）父组件：`onUnmounted(P)`  

4. **记忆主干规律**  
   - **挂载**：  
     - 逻辑准备：父 setup/render → 子 setup/render  
     - 完成挂载：**先子 mounted，后父 mounted**  
   - **更新**：  
     - `beforeUpdate`：先父后子  
     - `updated`：先子后父  
   - **卸载**：  
     - `beforeUnmount`：先父后子  
     - `unmounted`：先子后父  

理解这些顺序有助于在复杂父子组件通讯、依赖 DOM 存在的逻辑以及性能优化时，合理选择执行时机和注册对应生命周期钩子。
