## 🧩 题目

Vue 的响应式系统是如何进行「依赖收集」的？依赖更新时如何触发视图重新渲染？与 React 中的 `useEffect` 副作用机制相比，有哪些本质上的不同？

---

## 💡 解题思路

1. Vue：基于 `effect`、`track`、`trigger` 的自动依赖收集机制（按「访问的属性」建立依赖）
2. 依赖存储结构：`targetMap`（target → key → dep），更新时精准触发
3. React `useEffect`：显式依赖数组，按 render 执行和比较依赖
4. 对比两者：Vue 是「数据驱动依赖（拉模式）」、React 是「render 驱动副作用（推模式，显式依赖）」

---

## ✅ 面试回答（标准版）

1. **Vue 中的依赖收集核心过程（以 Vue 3 为例）**  
   - Vue 3 响应式系统的核心是一个 `effect` 函数，它包装了「需要在数据变化时重新执行的副作用逻辑」，比如组件的渲染函数：  

   ```js
   effect(() => {
     // 读取响应式数据
     console.log(state.count);
   });
   ```

   - 在执行 `effect` 时，框架会把当前正在运行的副作用函数记录在一个全局的 `activeEffect` 上  
   - 当我们访问响应式对象的某个属性时（在 `Proxy.get` 里），会调用 `track(target, key)`：  
     - 以 `target`（对象）和 `key`（属性名）为键，从 `targetMap` 里找到对应的依赖集合 `dep`（一个 `Set`）  
     - 把当前 `activeEffect` 加入 `dep` 中，这就是「依赖收集」  
   - 当我们修改这个属性时（在 `Proxy.set` 里），会调用 `trigger(target, key)`：  
     - 从 `targetMap` 中取出对应的 `dep`，遍历其中的 every `effect`，重新执行它们（通常通过调度器进行批量/异步执行）  

   简化的依赖存储结构可以理解为：

   ```js
   // WeakMap<target, Map<key, Set<effect>>>
   const targetMap = new WeakMap();
   ```

2. **视图更新流程**  
   - 组件的 render 函数本质上也是一个 `effect`：  
     - 首次执行 render 时，所有在其中访问的响应式属性都会对这个渲染 effect 进行依赖收集  
   - 当任意这些属性发生变化时，对应的 `trigger` 会把渲染 effect 放入更新队列  
   - 在下一轮微任务中批量执行更新队列 → 重新执行 render → diff 新旧虚拟 DOM → 最小化更新真实 DOM  

3. **与 React `useEffect` 的对比**  
   - **依赖声明方式不同**：  
     - Vue：依赖是「**自动收集的**」——在 effect 中访问了哪些响应式属性，这些属性就与该 effect 建立了依赖关系  
     - React：`useEffect` 需要显式传入依赖数组（`[a, b]`），React 会在每次 render 后比较新旧依赖的引用是否变化来决定是否执行 effect  
   - **触发时机不同**：  
     - Vue 的 effect（渲染 effect）主要在「数据变化 → 触发重新渲染」这条链路中  
     - React 的 `useEffect` 更偏向「渲染之后执行副作用」，通常用于订阅、DOM 操作、异步请求等，不直接驱动渲染本身（渲染逻辑在函数组件体内）  
   - **精度与心智模型**：  
     - Vue 通过按属性粒度的 `track/trigger` 实现了较为精细的依赖关系，避免了「忘记在依赖数组中写某个变量」的问题  
     - React 使用依赖数组则更显式，但容易出现依赖数组漏写或乱写导致的 bug，需要开发者对闭包和依赖关系有更清晰的认识  

4. **总结**  
   - Vue 的响应式系统是「**自动依赖收集 + 精准触发**」：根据在 effect 中「读取了哪些响应式属性」来建立依赖  
   - React 的 `useEffect` 更像是「**显式声明依赖 + 按 render 对比依赖**」的副作用模型  
   - 两者都服务于响应式 UI，但 Vue 更侧重「数据 → 视图」这条链路的细粒度依赖管理，而 React 的 `useEffect` 主要负责在渲染之后管理副作用和订阅。
