## 🧩 题目

在 Vue 中如何缓存组件？`keep-alive` 的使用场景和基本原理是什么？与之相关的生命周期钩子有哪些？

---

## 💡 解题思路

1. 说明 `keep-alive` 是内置抽象组件，用于缓存「动态组件」或「路由组件」
2. 使用方式：配合 `<component :is="...">` 或 `<router-view>`，常见属性 `include` / `exclude` / `max`
3. 生命周期：`activated` / `deactivated`，与普通 `mounted` / `unmounted` 区别
4. 简述原理：不销毁组件实例，而是从缓存池中取出/放回，复用 VNode/实例

---

## ✅ 面试回答（标准版）

1. **如何在 Vue 中缓存组件？**  
   - Vue 提供了内置的抽象组件 `<keep-alive>`，用来缓存**动态组件**或**路由组件**，避免频繁销毁和重建  

   ```vue
   <!-- 缓存动态组件 -->
   <keep-alive>
     <component :is="currentView" />
   </keep-alive>

   <!-- 缓存部分路由页面 -->
   <keep-alive include="User,Home">
     <router-view />
   </keep-alive>
   ```

   - 可以通过 `include` / `exclude`（支持字符串、正则、数组）控制哪些组件参与缓存，通过 `max` 控制最大缓存数量（超出后按 LRU 规则淘汰）  

2. **`keep-alive` 的典型使用场景**  
   - 多个页面/Tab 来回切换，希望**保留页面状态和滚动位置**，如：  
     - 列表页 -> 详情页 -> 返回列表页时，希望列表的筛选条件、翻页、滚动位置保持不变  
   - 复杂的表单页，不希望每次离开再回来都重新初始化表单数据和校验状态  
   - 频繁进入/退出的组件，初始化成本较高（如图表、富文本编辑器等）  

3. **相关的生命周期钩子**  
   - 普通组件：  
     - 首次挂载：`beforeMount` → `mounted`  
     - 销毁：`beforeUnmount` → `unmounted`（Vue3）  
   - 被 `keep-alive` 缓存的组件：  
     - 首次渲染时依然会触发 `mounted`  
     - 之后在激活/停用时不会再次 `mounted/unmounted`，而是触发：  
       - `activated`：从缓存中「激活」组件（再次显示）  
       - `deactivated`：组件被「停用」（从视图移除但实例仍在内存中）  
   - 在 Vue3 的组合式 API 中也可以通过 `onActivated` / `onDeactivated` 注册对应钩子  

4. **`keep-alive` 的实现原理（简化理解）**  
   - `keep-alive` 并不直接渲染 DOM，本质是一个**缓存容器**：  
     - 首次渲染某个子组件时，会正常创建组件实例并挂载，同时将该实例（VNode）存入一个缓存 Map 中，key 通常基于组件名和 `key` 属性  
     - 当该组件从视图中移除时，不会销毁实例，而是把对应的 VNode 标记为「缓存状态」，从 DOM 中移除但实例仍保留  
     - 当之后再次渲染相同的组件时，会从缓存中取出已有实例进行复用，而不是重新创建  
   - 这样可以保留组件内部的响应式状态、局部变量以及 DOM 状态（如输入框内容、滚动条位置等）  

5. **注意事项**  
   - 缓存过多组件会占用一定内存，需要合理使用 `include/exclude/max`  
   - 与路由守卫、全局状态配合时，要清楚「何时真正销毁组件」与「何时只是停用」  

一句话：`keep-alive` 通过对组件实例做「缓存和复用」来实现组件级别的状态保留，并通过 `activated/deactivated` 生命周期与普通 `mounted/unmounted` 区分开，是提升体验和性能的常用手段。
