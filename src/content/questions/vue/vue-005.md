## 🧩 题目

Vue 项目上线或本地运行时出现「白屏」的常见原因有哪些？你会如何快速定位问题并解决？

---

## 💡 解题思路

1. 分类：构建配置问题（资源路径、路由模式）、运行时 JS 报错、接口阻塞/首屏骨架缺失、服务端部署问题
2. 前端排查：控制台错误、网络请求、元素/样式、路由跳转
3. 构建与部署：`publicPath`、`history` 模式、Nginx 配置、资源 404
4. 预防措施：错误边界、loading/骨架屏、监控报警

---

## ✅ 面试回答（标准版）

Vue 项目白屏通常意味着：**首屏根组件没有成功渲染**，可能是构建/部署问题、JS 运行时错误或数据加载阻塞等原因共同导致。

1. **常见原因分类**  
   - **构建与部署配置问题**：
     - 打包后的静态资源路径错误，例如 `publicPath` / `base` 设置不当，导致 `app.js`、`chunk-vendors.js` 等加载 404  
     - 使用 `history` 路由模式但服务端没做「fallback 到 `index.html`」，刷新子路由直接 404  
   - **运行时 JS 错误**：
     - 根组件或路由组件中存在未捕获的异常（如访问 `undefined.xxx`），导致初始化阶段报错中断渲染  
     - 打包后某些依赖与运行环境不兼容（如老浏览器缺乏 Polyfill）  
   - **接口阻塞 / 数据依赖问题**：
     - 首屏渲染强依赖接口返回，但没做 loading/超时/错误兜底 → 用户感知为长时间白屏  
   - **样式问题**：
     - 根元素高度/背景色问题（例如 `#app` 容器高度为 0 或背景色为白色叠加），导致元素存在但视觉似乎是白屏  

2. **本地快速排查思路**  
   - 打开浏览器 DevTools：  
     - **Console**：查看是否有未捕获的 JS 报错（红色错误基本是首要线索）  
     - **Network**：确认主 JS/CSS 包和首屏接口是否成功加载或返回错误/超时  
     - **Elements**：检查 `#app` / 根组件是否已挂载，只是被样式隐藏或高度为 0  
   - 如果是路由导致的白屏，尝试访问根路径是否正常，再刷新子路由页面，看是否与后端路由配置有关  

3. **线上部署相关问题与解决方案**  
   - 静态资源 404 / 路径错：  
     - 检查打包时的 `publicPath`（Vue CLI）或 `base`（Vite）是否与部署路径一致，比如应用部署在 `/subapp/` 下  
   - history 模式导致刷新 404：  
     - 确保 Nginx/Apache 配置了「任意前端路由都回退到 `index.html`」，让前端路由系统接管  
   - 浏览器兼容性：  
     - 使用 `@vue/cli` 时开启 `babel` / polyfill，或在 Vite 项目中确认目标浏览器与构建目标一致  

4. **预防与优化**  
   - 提供基础的 **loading 状态或骨架屏**，避免用户看到纯白页面  
   - 对关键逻辑加上错误捕获（如 `try/catch`、错误边界组件、全局 `errorHandler`），避免因为单个组件错误导致整棵应用树白屏  
   - 在生产环境接入前端监控（如 Sentry、自建埋点），记录错误堆栈和白屏时间，便于追踪  

总结：排查白屏要从「资源是否加载成功 → JS 是否报错中断 → 路由/部署是否正确 → 样式和数据加载」这几条链路逐步排查，同时在工程上通过 loading、错误兜底和监控来降低白屏对用户的影响。
