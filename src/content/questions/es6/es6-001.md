## 🧩 题目

ES6 箭头函数和普通函数有什么区别？请从 `this` 指向、语法特性、使用限制等方面说明。

---

## 💡 解题思路

1. 语法更简洁：省略 `function`、`return`、参数括号等
2. `this` 绑定行为：箭头函数没有自己的 `this`，继承外层作用域
3. 不能作为构造函数、没有 `prototype`
4. 没有 `arguments`、不能用作生成器函数
5. 适用场景 vs 不适用场景

---

## ✅ 面试回答（标准版）

ES6 箭头函数与普通函数的核心区别主要体现在**语法形式、`this` 绑定方式以及能否作为构造函数**等几个方面：

1. **语法更简洁**  
   - 可以省略 `function` 关键字：`(a, b) => a + b`  
   - 当只有一个参数时可以省略小括号：`x => x * 2`  
   - 当函数体只有一行返回语句时可以省略 `return` 和大括号：`x => x + 1`  
   - 如果要返回对象字面量，需要用小括号包起来：`() => ({ a: 1 })`  

2. **没有自己的 `this`（词法绑定 `this`）**  
   - 普通函数的 `this` 是在**调用时**由调用方式决定的（谁调用就指向谁，或被 `call/apply/bind` 显式绑定）  
   - 箭头函数**不绑定自己的 `this`**，而是从定义时所在的外层作用域**词法捕获** `this`，无法再通过 `call/apply/bind` 改变  
   - 这使得它非常适合作为回调函数使用，避免在异步或事件回调中手动保存 `this`（比如 `const self = this` 或 `.bind(this)`）  

3. **不能作为构造函数**  
   - 箭头函数没有 `[[Construct]]` 内部方法，也没有 `prototype` 属性  
   - 因此**不能使用 `new` 调用箭头函数**，否则会抛出 TypeError  
   - 普通函数（包括 `function` 声明和函数表达式）可以作为构造函数使用，配合 `new` 创建实例对象  

4. **没有 `arguments` 对象**  
   - 普通函数内部可以通过 `arguments` 访问实参数组  
   - 箭头函数没有自己的 `arguments`，如果直接访问，会从外层作用域查找  
   - 在箭头函数中一般用 **rest 参数**（`...args`）来获取参数列表：`(...args) => {}`  

5. **不能用作生成器函数**  
   - 箭头函数不能与 `function*`、`yield` 搭配使用  
   - 也不能作为 `async function*` 等生成器的语法形式  

6. **适用与不适用场景**  
   - 适合：数组遍历、回调函数（如 `map/filter/reduce`）、Promise 链、事件回调中需要继承外层 `this` 的场景  
   - 不适合：需要 `this` 动态绑定的函数（如对象方法中希望根据调用方变化）、需要作为构造函数、需要使用 `arguments`、需要生成器功能等场景  

总结一下：箭头函数更适合声明**短小的函数和回调函数**，带来更简洁的语法和更易预测的 `this`，而普通函数依然在构造函数、原型方法、需要动态 `this` 绑定的场景下不可替代。
