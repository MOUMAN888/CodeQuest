## 🧩 题目

ES6 新增的 `Symbol` 基础数据类型有什么作用？它解决了哪些问题，适合用在什么场景？

---

## 💡 解题思路

1. `Symbol` 是新的原始类型，表示唯一值
2. 典型用途：对象的私有属性键，避免命名冲突
3. 内置 `Symbol`（如 `Symbol.iterator`）用于扩展语言协议
4. 对比字符串键的缺点，总结使用注意点

---

## ✅ 面试回答（标准版）

`Symbol` 是 ES6 新增的一种**原始数据类型**（primitive type），用来表示一个**唯一的标识符**。它的主要作用是：为对象添加独一无二的属性键，避免命名冲突，同时配合一些内置 `Symbol` 实现语言层面的协议扩展。

1. **唯一性与安全性**  
   - 通过 `Symbol()` 创建的值是唯一的，即使描述字符串相同也不会相等：

   ```js
   const a = Symbol('id');
   const b = Symbol('id');
   console.log(a === b); // false
   ```

   - 因此非常适合作为对象属性的键，避免与其他属性发生冲突（尤其是在多人协作或引入第三方库时）  

2. **作为对象的私有属性键**  
   - 传统对象的键一般使用字符串，很容易「撞名」：

   ```js
   const obj = {};
   const KEY = Symbol('internal');
   obj[KEY] = 123; // 外部不太容易无意访问/覆盖
   ```

   - 使用 `Symbol` 作为键的属性不会出现在普通的 `for...in`、`Object.keys()` 遍历中，需要通过 `Object.getOwnPropertySymbols()` 或 `Reflect.ownKeys()` 才能获取  
   - 这使得 `Symbol` 属性更接近「半私有」的效果  

3. **内置 `Symbol` 与语言协议**  
   ES6 定义了一系列内置的 `Symbol`，用于自定义一些语言内部行为，例如：  
   - `Symbol.iterator`：定义对象的默认迭代器，使其可用于 `for...of`  
   - `Symbol.toStringTag`：自定义 `Object.prototype.toString.call(obj)` 的结果  
   - `Symbol.hasInstance`：自定义 `instanceof` 行为  
   - `Symbol.toPrimitive`：定义对象被转为原始类型时的行为  

   通过实现这些内置 `Symbol` 属性，我们可以优雅地扩展对象与语言内建机制之间的协作方式。  

4. **注意点**  
   - `Symbol` 不能与字符串直接拼接（会抛错），如需输出可以显式调用 `String(sym)` 或 `sym.description`  
   - 一般不用于序列化（`JSON.stringify` 会忽略 `Symbol` 键和 `Symbol` 值）  
   - 当需要全局共用同一个 `Symbol` 时，可以使用 `Symbol.for('key')` / `Symbol.keyFor()` 的全局注册表  

总结：`Symbol` 的核心价值在于「唯一性」和「协议扩展」。在工程实践中，既可以用其作为对象的内部/私有属性键，避免污染；也可以通过内置 `Symbol` 来实现可迭代对象、定制类型标签等高级能力。
