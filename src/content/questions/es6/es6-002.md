## 🧩 题目

ES6 箭头函数能当构造函数使用吗？如果不能，请说明原因。

---

## 💡 解题思路

1. 构造函数的本质：通过 `new` 调用、内部有 `[[Construct]]`
2. 箭头函数的内部特性：没有 `[[Construct]]` 和 `prototype`
3. 说明使用 `new` 调用箭头函数时会抛出什么错误
4. 补充说明适合当构造函数的是哪类函数

---

## ✅ 面试回答（标准版）

**箭头函数不能当构造函数使用。**  

1. **从规范角度说明原因**  
   - 能否作为构造函数，取决于函数对象是否拥有内部方法 `[[Construct]]`  
   - 传统的 `function` 声明/表达式创建的函数有 `[[Construct]]`，因此可以通过 `new` 调用  
   - 箭头函数在 ES6 规范中被设计为**不可构造**（non-constructible），没有 `[[Construct]]` 内部方法  
   - 同时它也**没有 `prototype` 属性**（或其原型不用于实例继承），这与构造函数的设计不符  

2. **实际表现**  
   - 当你尝试 `new` 一个箭头函数时，例如：

   ```js
   const Foo = () => {};
   const f = new Foo(); // ❌
   ```

   - 运行会抛出类似 `TypeError: Foo is not a constructor` 的错误  

3. **正确的做法**  
   - 需要构造函数时，应使用普通函数或 ES6 `class`：

   ```js
   function Person(name) {
     this.name = name;
   }

   class Animal {
     constructor(name) {
       this.name = name;
     }
   }
   ```

总结：箭头函数主要用于声明普通函数和回调函数，它**不能作为构造函数**，也不应该出现在需要原型继承或实例化的场景中。
