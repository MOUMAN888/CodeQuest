## 🧩 题目

如何计算 CSS 的优先级（specificity）？请说明不同选择器之间的优先级比较规则。

---

## 💡 解题思路

1. 理解「就近原则」与「优先级」：!important > 内联样式 > 选择器优先级 > 源顺序
2. 记住优先级四元组（从高到低）：行内样式、ID、类/属性/伪类、元素/伪元素
3. 会用简单的数字模型表示：`(a, b, c, d)` 或 `0,1,0,0` 等
4. 掌握几个易混场景：多个类 vs 一个 ID、`!important` 的作用和副作用

---

## ✅ 面试回答（标准版）

CSS 冲突时最终生效的规则由**来源 + 优先级 + 书写顺序**共同决定，其中最核心的是「选择器优先级」（specificity）。通常可以用一个四元组表示：  
`(a, b, c, d)`，从左到右优先级依次降低：

1. **优先级来源的大致顺序**  
   - **最高**：`!important` 声明  
   - 其次：行内样式（`style` 属性）  
   - 再次：根据「选择器优先级」比较  
   - 如果优先级相同：后写的覆盖先写的（就近原则，按源代码顺序）  

2. **选择器优先级的计算模型**（常用记忆方式）  
   可以记成：`行内样式 > ID > 类/属性/伪类 > 元素/伪元素`：
   - `a`：是否包含行内样式（一般作为单独一层理解）  
   - `b`：ID 选择器数量（如 `#app`）  
   - `c`：类选择器、属性选择器、伪类选择器数量  
     - 类：`.btn`、`.active`  
     - 属性：`[type="text"]`、`[disabled]`  
     - 伪类：`:hover`、`:focus`、`:not()`、`:nth-child()` 等  
   - `d`：元素选择器、伪元素选择器数量  
     - 元素：`div`、`p`  
     - 伪元素：`::before`、`::after`  

   示例：  
   - `div` → `(0, 0, 0, 1)`  
   - `.box.active` → `(0, 0, 2, 0)`  
   - `#app .box` → `(0, 1, 1, 0)`  
   - `a:hover` → `(0, 0, 1, 1)`  
   比较时，从左到右逐位比较，谁大谁优先。  

3. **几个典型对比**  
   - `#id` 一般会**压过**多个类选择器：`#id` > `.a.b.c`  
   - 行内样式（如 `<div style="color: red">`）的优先级会高于任何普通选择器  
   - 加了 `!important` 的声明会覆盖正常优先级下的所有规则（除非对方也加了 `!important` 且优先级更高）  

4. **实践中的建议**  
   - 尽量使用**类选择器 + 扁平结构**来组织样式，避免 ID、大量嵌套、过度依赖 `!important`  
   - 当需要覆盖第三方库样式时，可适当提升选择器权重或局部使用 `!important`，并配合更明确的作用域（如包裹容器类名）  

总结来说：先比较是否有 `!important`，再看行内样式，然后用「ID > 类/属性/伪类 > 元素/伪元素」的规则算出具体优先级，如果还相同则后写的胜出。
