## 🧩 题目

`display: none` 与 `visibility: hidden` 有什么区别？在实际项目中应该如何选择？

---

## 💡 解题思路

1. 文档流：是否占据空间  
2. 交互性：是否还能响应事件  
3. 渲染与回流：对布局计算的影响  
4. 典型使用场景对比

---

## ✅ 面试回答（标准版）

`display: none` 和 `visibility: hidden` 都能让元素看不见，但它们在布局和交互上的行为有明显差异：

1. **是否参与文档流**  
   - `display: none`：元素**直接从文档流中移除**，不再占据任何空间，就像没有这个元素一样，周围元素会重新排布  
   - `visibility: hidden`：元素**仍然占据原来的空间**，只是内容不可见，布局不会重新计算  

2. **是否可交互**  
   - `display: none`：元素不可见，也不会响应任何鼠标事件或键盘事件  
   - `visibility: hidden`：大多数实现中也不会响应鼠标点击等交互，但仍然占着那块区域，可能影响布局或命中测试  

3. **渲染性能与回流**  
   - `display: none` → 改为可见（如 `display: block`）时，浏览器需要重新把它插入文档流，进行一次完整的布局计算（回流）  
   - `visibility: hidden` → 设为 `visible` 时，只是改变可见性，布局结构本身不变，相对来说更「轻量」一些  

4. **典型使用场景**  
   - 使用 `display: none`：  
     - Tab 切换中隐藏未激活的面板  
     - 条件渲染（不需要元素占位时）  
     - 弹窗/抽屉完全关闭时  
   - 使用 `visibility: hidden`：  
     - 需要保留占位，避免布局抖动  
     - 某些场景下做「内容预留」（例如表头对齐等）  

总结：如果你希望元素「彻底从布局中消失」，用 `display: none`；如果你希望它只是「透明不可见，但布局不变」，用 `visibility: hidden`。在做交互动效或复杂布局时，二者的选择会直接影响页面的回流、重绘和用户体验。
