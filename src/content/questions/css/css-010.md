## 🧩 题目

常见的 CSS 布局单位有哪些？请简单介绍它们的含义和典型使用场景。

---

## 💡 解题思路

1. 绝对长度：`px`
2. 相对长度：`%`、`em`、`rem`
3. 视口单位：`vw`、`vh`、`vmin`、`vmax`
4. 其他布局相关单位：`fr`（Grid）、`ch` 等

---

## ✅ 面试回答（标准版）

CSS 中有很多不同的单位，用来表达尺寸、间距和布局比例等。常见布局相关单位可以分为几类：

1. **绝对长度单位**  
   - **`px`（像素）**：最常用的单位，表示一个设备像素（在现代浏览器中通常是逻辑像素，已经过 DPI 抽象）  
   - 特点：**稳定可控**，不会随父元素或视口变化而变化，适合像素级还原的场景，如边框、图标大小等  

2. **相对父元素/自身的单位**  
   - **百分比 `%`**：相对于父元素的某个尺寸：
     - 宽度 `width: 50%` 通常相对于包含块的宽度  
     - 高度则要看具体属性定义，有些情况与包含块高度有关  
     - 常用于自适应布局（如两栏 30%/70%）  
   - **`em`**：相对于当前元素的字体大小（`font-size`）  
     - `padding: 2em` 表示为当前字体大小的 2 倍  
     - 会随层级叠加，容易出现「越嵌套越大」的问题  
   - **`rem`**：相对于根元素（`html`）的字体大小  
     - 通常配合 `html { font-size: 16px; }` 做响应式缩放  
     - 优点是整个站点易于做「一处调节，全局生效」的字号/间距缩放  

3. **视口单位（Viewport Units）**  
   - **`vw`**：视口宽度的 1%  
   - **`vh`**：视口高度的 1%  
   - **`vmin`**：视口宽高中的较小值的 1%  
   - **`vmax`**：视口宽高中的较大值的 1%  
   - 适合做：全屏背景、首屏 Banner、需要随窗口变化的字号或块级区域高度，如 `height: 100vh` 的满屏页面  

4. **布局专用单位**  
   - **`fr`（fraction）**：Grid 布局中的「份数单位」，表示网格容器中剩余空间的分配比例  
     - 比如 `grid-template-columns: 1fr 2fr;` 表示两列分别占剩余空间的 1/3 与 2/3  
   - **`ch`**：大致表示「0」字符的宽度，用于与字符宽度相关的布局（如输入框宽度）  

在实际开发中，一般会**混合使用**这些单位：用 `px` 控制细节，用 `%` 和 `fr` 控制比例布局，用 `rem` 做全局可调的间距/字号，让页面在不同设备和缩放条件下都能保持合理的观感。
