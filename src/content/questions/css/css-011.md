## 🧩 题目

CSS 中 `flex-grow` 和 `flex-shrink` 分别有什么作用？Flex 布局的核心属性有哪些？主轴方向上尺寸是如何计算和分配的？

---

## 💡 解题思路

1. 回顾 Flex 布局：容器属性与项目属性
2. 解释 `flex-grow`：有剩余空间时按比例放大
3. 解释 `flex-shrink`：空间不足时按比例收缩
4. 简述 `flex-basis` 与简写 `flex`，以及主轴方向上的计算过程

---

## ✅ 面试回答（标准版）

1. **Flex 布局概览**  
   - 设置 `display: flex` 或 `inline-flex` 的元素称为「Flex 容器」，其直接子元素是「Flex 项目」  
   - 容器常用属性：`flex-direction`、`flex-wrap`、`justify-content`、`align-items`、`align-content` 等  
   - 项目常用属性：`flex`、`flex-grow`、`flex-shrink`、`flex-basis`、`align-self` 等  

2. **`flex-grow`：有剩余空间时如何放大**  
   - 表示「**在主轴方向上，项目可以分配剩余空间的放大比例**」  
   - 默认值为 `0`，表示不放大  
   - 若多个项目 `flex-grow` 不同，则根据它们的 `flex-grow` 比例分配剩余空间：  

   ```css
   .item1 { flex-grow: 1; }
   .item2 { flex-grow: 2; }
   ```

   - 剩余空间将被分成 3 份，`item1` 分 1 份，`item2` 分 2 份  

3. **`flex-shrink`：空间不足时如何收缩**  
   - 表示「**在主轴方向空间不足时，项目按什么比例收缩**」  
   - 默认值为 `1`，表示允许收缩；设为 `0` 可禁止该项收缩  
   - 当总宽度（或高度）超出容器时，浏览器会根据每个项目的 `flex-shrink` 与其基础尺寸计算收缩量，`flex-shrink` 越大收缩得越多  

4. **`flex-basis` 与 `flex` 简写**  
   - `flex-basis`：主轴方向上的「初始基础尺寸」，类似于 `width`/`height` 的初始值  
   - 完整形态：`flex: flex-grow flex-shrink flex-basis;`，例如：

   ```css
   .item { flex: 1 1 200px; }
   ```

   - 常见简写：  
     - `flex: 1;` 等价于 `flex: 1 1 0;`  
     - `flex: none;` 等价于 `flex: 0 0 auto;`  

5. **主轴方向的尺寸计算流程（简化理解）**  
   - 第一步：根据 `flex-basis`（或内容/width）计算每个项目的基础尺寸之和  
   - 第二步：与容器主轴尺寸比较，得出「剩余空间」或「超出空间」  
   - 第三步：  
     - 若有剩余空间 → 按 `flex-grow` 比例在项目之间分配  
     - 若空间不足 → 按 `flex-shrink` 比例在项目之间压缩  

总结：`flex-grow` 控制「空间富余时谁长得多」，`flex-shrink` 控制「空间不足时谁缩得多」，配合 `flex-basis` 和容器的主轴设置，构成了 Flex 布局的核心尺寸计算规则。
