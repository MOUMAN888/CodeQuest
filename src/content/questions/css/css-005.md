## 🧩 题目

利用 CSS 隐藏元素的方法有哪些？它们之间有什么区别，适用在什么场景？

---

## 💡 解题思路

1. 常见方式：`display: none`、`visibility: hidden`、`opacity: 0`
2. 视觉上移出视口/裁剪：`position` + 移动、`clip-path`/`clip` 等
3. 语义与辅助功能层面：既隐藏视觉又给屏幕阅读器使用 vs 完全对外不可见
4. 重点对比是否占据文档流、是否可点击、是否会被读屏器读到

---

## ✅ 面试回答（标准版）

用 CSS 隐藏元素不仅有多种写法，而且不同写法在「是否占据空间」「是否可交互」「无障碍（A11y）」等方面差异很大。常见方式包括：

1. **`display: none`**  
   - 元素从文档流中移除，不再占据空间  
   - 不会响应点击、hover 等事件（除非通过 JS 直接操作 DOM）  
   - 对大多数读屏器来说，相当于不存在  
   - 常用于：Tab 切换面板、弹窗隐藏、条件渲染等 **完全不需要出现** 的场景  

2. **`visibility: hidden`**  
   - 元素仍然占据文档流中的空间，但内容不可见  
   - 不会响应鼠标点击、hover 等交互  
   - 通常也不会被读屏器读出（视具体实现而定）  
   - 适合：**需要保留布局占位**，但暂时不显示的场景，例如表格行临时隐藏等  

3. **`opacity: 0`**  
   - 元素仍在文档流中，占据空间  
   - 仍然可以触发事件（如点击、hover），只是完全透明  
   - 通常仍会被读屏器读到  
   - 适合：需要过渡动画（如淡入淡出）或希望「视觉上消失但仍可交互」的场景，要注意不要制造「看不见但挡操作」的问题  

4. **通过定位/裁剪隐藏**  
   - 如：`position: absolute; left: -9999px;` 把元素移到可视区域外  
   - 或使用 `clip` / `clip-path` / `transform: scale(0)` 等方式将其裁剪/缩放到不可见  
   - 常用于无障碍场景：为读屏器保留内容，但不在屏幕上可见（如「跳转到内容」链接）  

5. **其他与「隐藏」相关的属性**  
   - `overflow: hidden`：隐藏超出容器的内容，但容器本身仍可见  
   - `pointer-events: none`：不再响应鼠标事件，但视觉上仍可见  

综合来看，如果只是简单「不想显示」元素，大多数情况下会使用 `display: none`；如果想保留占位则可以考虑 `visibility: hidden`；如果需要动画或保留交互，则会用 `opacity: 0` 配合过渡。在需要兼顾无障碍时，还要结合 ARIA 属性和适当的隐藏方式来区分「视觉隐藏」和「对所有用户都隐藏」。
