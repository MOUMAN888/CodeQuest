## 🧩 题目

如何用 CSS 实现元素的淡入淡出效果？`transition` 和 `animation` 有什么区别？分别适用于哪些场景？

---

## 💡 解题思路

1. 使用 `opacity` + `transition` 实现淡入淡出（配合 `display`/`visibility`）
2. 使用 `@keyframes` + `animation` 实现自动循环或更复杂控制
3. 对比：`transition` 依赖状态切换、只能定义起始/结束；`animation` 可定义关键帧、可自动播放/循环
4. 实战中如何封装淡入淡出 class

---

## ✅ 面试回答（标准版）

1. **用 `transition` 实现淡入淡出（最常用）**  

   HTML：

   ```html
   <div class="fade" id="box"></div>
   ```

   CSS：

   ```css
   .fade {
     opacity: 0;
     transition: opacity 0.4s ease;
   }

   .fade.show {
     opacity: 1;
   }
   ```

   JS 触发：

   ```js
   const box = document.getElementById('box');
   box.classList.add('show');   // 淡入
   box.classList.remove('show'); // 淡出
   ```

   - 通过修改类名改变 `opacity`，`transition` 会对属性变化进行平滑过渡  
   - 若需要完全移除交互/布局影响，可以再配合 `visibility` 或 `pointer-events` 使用  

2. **用 `animation` + `@keyframes` 实现淡入淡出**  
   - 适合自动播放、循环播放或更复杂的时间控制场景：

   ```css
   @keyframes fadeIn {
     from { opacity: 0; }
     to   { opacity: 1; }
   }

   .fade-in {
     animation: fadeIn 0.4s ease forwards;
   }
   ```

   - 若要实现「先淡入、停顿、再淡出」，可以在 `@keyframes` 中定义多个关键帧，而 transition 不好表达这种多阶段变化  

3. **`transition` vs `animation` 的区别**  
   - **触发方式**：
     - `transition`：需要一个**状态变化**（如类名变化、行内样式变化）来触发，只定义「从 A 到 B」的过渡  
     - `animation`：可在元素出现时自动播放，也可通过类名/JS 控制何时开始  
   - **表达能力**：
     - `transition`：简洁、适合简单的「进入/离开」效果  
     - `animation`：通过 `@keyframes` 支持多个中间关键帧，可以实现复杂的时间曲线与多阶段动画  
   - **循环与控制**：
     - `transition`：不支持循环，只在属性变化时执行一次  
     - `animation`：可以设置 `animation-iteration-count`（如 `infinite`）、`direction`、`delay` 等  

4. **实践建议**  
   - 简单的显隐、hover 效果：优先使用 `transition + opacity/transform`  
   - 复杂的逐帧动画、强调型动效（如 loading、骨架屏）：使用 `animation + @keyframes`  

一句话：淡入淡出最常见做法是用 `opacity` 配合 `transition`，而 `animation` 更适合需要多阶段、可循环的复杂动画。
