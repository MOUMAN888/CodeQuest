## 🧩 题目

什么是「受控组件」和「非受控组件」？请分别举例说明在什么情况下更适合使用哪一种。

---

## 💡 解题思路

1. 受控组件：表单值完全由 React state 控制
2. 非受控组件：使用原生 DOM 状态，通过 `ref` 读取
3. 受控组件适合需要即时校验、联动展示、可预览的场景
4. 非受控适合简单表单、性能要求高或需要快速迁移老代码

---

## ✅ 面试回答（标准版）

在 React 中，如果一个表单元素的值完全由组件 state 决定，并通过 `value={state}` + `onChange={setState}` 这种方式双向绑定，我们称之为**受控组件**；反之，如果表单元素的值主要由 DOM 自己维护，只是在需要的时候通过 `ref.current.value` 读取或重置，就叫做**非受控组件**。  
受控组件的优点是：所有表单状态都在 React 里，可方便地做实时校验、禁用提交、联动展示、保存草稿等操作，但也会带来更多的渲染和状态更新；非受控组件则更接近原生表单，逻辑简单、性能开销小，适合对交互要求不高或只在提交时统一收集数据的场景。实际项目里，复杂表单往往使用受控组件配合表单库，而一些简单搜索框、文件上传等则可以考虑用非受控组件简化实现。
