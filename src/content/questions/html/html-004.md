## 🧩 题目

HTML 中 `src` 和 `href` 的区别是什么？它们各自的加载机制如何？分别适用于哪些场景？

---

## 💡 解题思路

1. 概念区分：`src` 是「嵌入、替换当前内容」，`href` 是「指向、链接一个资源」
2. 场景：`src` 常用于 `<img>`、`<script>`、`<iframe>`；`href` 用于 `<a>`、`<link>` 等
3. 加载机制：`src` 会阻塞（如脚本执行前阻塞解析），`href` 的样式/资源可以并行加载
4. 对 SEO、可访问性和性能的影响（简要提及）

---

## ✅ 面试回答（标准版）

在 HTML 中，`src` 和 `href` 的核心区别在于：**`src` 用来「嵌入」外部资源，`href` 用来「指向」外部资源**。

1. **`src`（source）的语义与使用场景**  
   - 含义：指定外部资源的位置，将其**嵌入到当前标签所在的位置**，用这个资源「替换」标签本身内容  
   - 常见于：  
     - `<img src="...">`：嵌入图片资源  
     - `<script src="..."></script>`：嵌入并执行外部 JS  
     - `<iframe src="..."></iframe>`：嵌入一个子页面  
   - 加载机制与影响：  
     - 对于 `<script src>`，在默认情况下会**阻塞 HTML 解析**，直到脚本下载并执行完成（除非加上 `defer` / `async`）  
     - `<img src>` 等资源不会阻塞解析，但如果位于首屏区域，会影响首屏渲染的整体体验  

2. **`href`（hyper reference）的语义与使用场景**  
   - 含义：指定当前元素所**指向的资源地址**，表示一种「链接关系」  
   - 常见于：  
     - `<a href="...">`：超链接，点击后跳转到目标地址  
     - `<link rel="stylesheet" href="...">`：告诉浏览器「这里有一个外部样式表」，浏览器会并行去加载  
   - 加载机制与影响：  
     - `<link rel="stylesheet">` 不会阻塞 HTML 解析本身，但**会影响样式的应用和首屏渲染**（浏览器通常会在样式就绪后再绘制）  
     - `<a href>` 不会主动加载目标页面内容，只有在用户点击或脚本触发跳转时才会导航  

3. **总结与实践建议**  
   - **需要将外部资源「嵌入」到当前文档中时，用 `src`**，比如图片、脚本、内嵌页面等  
   - **需要声明「跳转关系」或「关联资源」时，用 `href`**，比如超链接、样式表等  
   - 对脚本与样式的加载时机，要结合性能和渲染需求合理使用 `defer`、`async`、`preload` 等属性进行优化  

一句话：`src` 更像是「把这个资源拿来用在这里」，`href` 更像是「这个元素指向那边的一个资源或页面」。
